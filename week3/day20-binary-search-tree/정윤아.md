## Q1. BST에서 탐색이 평균 O(log n)인 이유는?

[답변]
* BST: Binary Search Tree
  - 왼쪽 서브트리에는 현재 노드보다 작은 값
  - 오른쪽 서브트리에는 현재 노드보다 큰 값
  - 탐색: 매 단계마다 현재 노드와 값 비교 -> 왼쪽/오른쪽 서브트리만 선택
  - 트리가 평균적으로 균형 -> 한 번 비교할 때마다 탐색 범위가 절반으로 줄어듦
  - 트리 높이: 대체로 logn 수준 -> 탐색 시간 평균적으로 O(logn)
    - 탐색 비용은 트리의 높이에 비례
    - 균형 잡힌 이진 트리 높이: 대략 log2n

[보완]
* 탐색 비용이 트리 높이에 비례하는 이유
  - 루트 -> 아래로 한 경로만 따라 내려감
  - 왼쪽 < 현재 노드 < 오른쪽
  - 가야 할 방향이 왼쪽 or 오른쪽 하나로 정해짐
  - 나머지 반 제외

## Q2. BST의 삭제에서 자식이 2개인 노드를 삭제할 때 처리 방법은?

[답변]
* 노드 자식 2개 -> 왼쪽, 오른쪽 서브트리 간의 정렬 관계 유지 필요 -> 대체값 찾기
  - 두 방법 중 하나
    - 오른쪽 서브트리에서 가장 작은 값 찾음 (중위 후속자)
      - 실무에 많이 사용
    - 왼쪽 서브트리에서 가장 큰 값 찾음 (중위 선행자)
  - 찾은 대체 노드의 값 -> 삭제 노드에 복사
  - 대체 노드는 BST 성질상 자식이 최대 1개
    - 리프 노드거나 자식이 한 개일 것!
    - 각 서브트리에서의 극단값이기 때문
  => 삭제가 쉬워짐: 자식이 하나면 삭제한 자리에 그 자식을 연결

## Q3. BST가 편향 트리가 되면 시간복잡도가 O(n)이 되는 이유는?

[답변]
* 편향 트리
  - 노드들이 한쪽으로만 계속 연결
  - 사실상 연결 리스트
  - 높이가 n에 가까워짐
  - BST의 탐색/삽입/삭제 모두 트리의 높이에 비례
    => 최악의 경우 O(n)

[보완]
* 탐색: 찾는 값이 나올 때까지 내려감 - 한 레벨씩만 이동 => 비용: 높이에 비례
* 삽입: 값을 비교하며 들어갈 위치 찾을 때까지 내려감 => 그 자리에 새 노드 붙임
  - 위치 찾는 과정: 높이에 비례
* 삭제: 삭제할 노드를 찾기 위해 먼저 아래로 내려감 - 후속자/선행자 찾을 때도 서브트리 안에서 아래로 내려감: 높이에 비례

## Q4. AVL 트리는 어떻게 균형을 유지하는가? 회전의 종류는?

[답변]
* AVL 트리: Adelson-Velsky, Vandis, Landis
  - 각 노드의 왼쪽 서브트리와 오른쪽 서브트리 높이 차이 관리 -> 균형 유지
  - 높이 차이: 균형 인수 => 보통 -1, 0, 1 범위 내
  - 삽입/삭제 후 노드 균형 깨짐 -> 회전 연산 -> 트리 구조 재조정
  - 회전 종류
    - LL 회전: 왼쪽-왼쪽으로 치우친 경우 -> 단일 오른쪽 회전
    - RR 회전: 오른쪽-오른쪽으로 치우친 경우 -> 단일 왼쪽 회전
    - LR 회전: 왼쪽-오른쪽으로 치우친 경우 -> 왼쪽 회전(왼쪽-왼쪽으로 변형) 후 오른쪽 회전
    - RL 회전: 오른쪽-왼쪽으로 치우친 경우 -> 오른쪽 회전(오른쪽-오른쪽응로 변형) 후 왼쪽 회전

[보완]
* 왼쪽 회전
  - 어떤 노드 X의 오른쪽 자식 Y가 위로 올라오고, X는 Y의 왼쪽 자식 자리로 내려가는 연산
  - 부모가 더 작은 경우
* 오른쪽 회전
  - 어떤 노드 Y의 왼쪽 자식 X가 위로 올라오고, Y는 X의 오른쪽 자식 자리로 내려가는 연산
  - 부모가 더 큰 경우

## Q5. Red-Black 트리의 특성은 무엇인가?

[답변]
* Red-Black 트리
  - 각 노드에 빨간색 또는 검은색 부여 -> 균형 유지
  - 자기 균형 BST

* 주요 특성
  - 루트 노드: 검은색
  - 리프 노드(NUL, null leaf): 검은색
  - 빨간색 노드의 자식은 검은색
  - 어떤 노드에서 출발하더라도, 그 노드에서 리프까지 가는 모든 경로의 검은색 노드 수는 같음
  - 새 노드는 무조건 빨간색
    - Double Red 발생 시
      1. 삼촌 노드가 검정색: Restructuring
      2. 삼촌 노드가 빨간색: Recoloring

## Q6. AVL 트리와 Red-Black 트리 중 어떤 것을 선택해야 하는가?

[답변]
* AVL 트리
  - 탐색 성능이 더 중요한 경우
  - 엄격하게 균형 맞춤 -> 트리 높이가 더 낮게 유지되는 경향
* Red-Black 트리
  - 삽입 / 삭제가 자주 일어나는 경우
  - 균형 조건이 덜 엄격 -> 회전 / 재조정 비용이 비교적 적음
