## 1. BST에서 탐색이 평균 **O(log n)** 인 이유

**BST (Binary Search Tree)** 는 다음 성질을 가진다.

-   왼쪽 서브트리의 모든 값 < 현재 노드
-   오른쪽 서브트리의 모든 값 > 현재 노드

따라서 탐색 과정은 다음과 같다.

```
현재 노드와 비교
→ 작으면 왼쪽 이동
→ 크면 오른쪽 이동
```

### 높이 관점에서 보면

균형 잡힌 BST라면 높이는

[
h \approx \log_2 n
]

왜냐하면

```
level 0 : 1
level 1 : 2
level 2 : 4
level 3 : 8
...
```

즉

[
2^h \approx n
]

따라서

[
h \approx \log_2 n
]

탐색 시 **루트 → 리프까지 내려가므로**

```
탐색 시간 = 트리 높이 = O(log n)
```

---

## 2. BST에서 **자식이 2개인 노드 삭제**

삭제는 **3가지 경우**가 있다.

### 1️⃣ 자식 없음 (leaf)

그냥 삭제

```
   5
  /
 3   → 3 삭제
```

---

### 2️⃣ 자식 1개

자식을 부모와 연결

```
   5
  /
 3
  \
   4

3 삭제 → 4가 5의 왼쪽
```

---

### 3️⃣ 자식 2개 (가장 중요한 케이스)

삭제할 노드를 **대체할 노드**가 필요하다.

대체 후보는 두 가지

```
1️⃣ Inorder successor
   → 오른쪽 서브트리의 최소값

2️⃣ Inorder predecessor
   → 왼쪽 서브트리의 최대값
```

보통 **successor**를 사용한다.

### 예시

```
        50
       /  \
     30    70
           / \
         60   80
```

50 삭제

```
successor = 60
```

변환

```
        60
       /  \
     30    70
              \
               80
```

절차

1. successor 찾기
2. 삭제할 노드 값을 successor로 교체
3. successor 노드 삭제

successor는 **최대 자식 1개**만 가진다 → 삭제 쉬움

## 3. BST가 편향 트리가 되면 **O(n)** 인 이유

다음과 같이 삽입하면

```
1
 \
  2
   \
    3
     \
      4
       \
        5
```

이건 사실상

```
Linked List
```

높이

```
h = n
```

탐색 과정

```
1 → 2 → 3 → 4 → 5
```

따라서

```
탐색 = O(n)
삽입 = O(n)
삭제 = O(n)
```

즉 **BST의 성능은 트리 높이에 의존한다.**

## 4. AVL 트리는 어떻게 균형을 유지하는가

AVL 트리는 **Self-balancing BST** 이다.

각 노드는 **Balance Factor** 를 가진다.

[
BalanceFactor = height(left) - height(right)
]

AVL 조건

```
|balance factor| ≤ 1
```

즉

```
-1, 0, 1 만 허용
```

이를 벗어나면 **회전(rotation)** 으로 균형을 맞춘다.

---

### AVL 회전 종류

총 **4가지**

### 1️⃣ LL Rotation

```
    z
   /
  y
 /
x
```

→ **Right Rotation**

```
    y
   / \
  x   z
```

---

### 2️⃣ RR Rotation

```
z
 \
  y
   \
    x
```

→ **Left Rotation**

```
    y
   / \
  z   x
```

---

### 3️⃣ LR Rotation

```
    z
   /
  y
   \
    x
```

→

```
Left rotation (y)
Right rotation (z)
```

---

### 4️⃣ RL Rotation

```
z
 \
  y
 /
x
```

→

```
Right rotation (y)
Left rotation (z)
```

---

### AVL 특징

```
높이 ≤ 1.44 log₂ n
```

그래서

```
search = O(log n)
insert = O(log n)
delete = O(log n)
```

---

## 5. Red-Black 트리의 특성

Red-Black Tree는 **느슨한 균형 BST**이다.

각 노드는

```
RED
BLACK
```

색을 가진다.

### 규칙

1️⃣ 모든 노드는 **red or black**

2️⃣ 루트는 **black**

3️⃣ **red 노드의 자식은 black**
(연속 red 금지)

4️⃣ **모든 leaf(null) 는 black**

5️⃣ 어떤 노드에서 leaf까지 가는 **black node 수 동일**

이를 **Black Height** 라고 한다.

---

### 높이 보장

Red-Black 트리 높이

[
h \le 2 \log_2(n+1)
]

따라서

```
search = O(log n)
insert = O(log n)
delete = O(log n)
```

---

## 6. AVL vs Red-Black 트리 선택

| 특징      | AVL       | Red-Black       |
| --------- | --------- | --------------- |
| 균형      | 매우 엄격 | 느슨            |
| 탐색 속도 | 더 빠름   | 조금 느림       |
| 삽입/삭제 | 회전 많음 | 회전 적음       |
| 높이      | 더 낮음   | 조금 높음       |
| 구현      | 복잡      | 상대적으로 단순 |

---

### 선택 기준

#### AVL 트리

```
읽기 많음
검색 성능 중요
```

예

```
DB 인덱스
메모리 검색 구조
```

---

#### Red-Black 트리

```
삽입/삭제 많음
동적 구조
```

그래서 **실무에서 더 많이 사용**

대표 사례

```
Java TreeMap
Java TreeSet
C++ std::map
Linux kernel scheduler
```
