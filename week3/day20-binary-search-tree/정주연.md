## 🧠 학습 질문

## BST에서 탐색이 평균 O(log n)인 이유는?

- 균형이 적당히 잡힌 BST는 각 노드가 배열의 이분 탐색처럼 역할
- 한 번 비교할 때마다 <왼쪽 or 오른쪽 서브트리>로 내려가면서 검색 범위가 약 절반씩 줄어듦
- 트리 높이가 대략 log⁡n 이고, 루트에서 리프까지 가도 최대 그 높이만큼 비교하면 되므로 평균 시간복잡도가 O(log n)

## BST의 삭제에서 자식이 2개인 노드를 삭제할 때 처리 방법은?

- 노드의 오른쪽 서브트리에서 최소값(가장 왼쪽 노드) 을 찾음
    
    (또는 왼쪽 서브트리에서 최대값)
    
- 그 값을 삭제 대상 노드에 복사
- 실제로는 오른쪽 서브트리의 최소값 노드(자식이 0~1개인 경우)를 삭제

→ BST의 정렬 순서(L→root→R)를 깨지지 않음

## BST가 편향 트리가 되면 시간복잡도가 O(n)이 되는 이유는?

- 편향 트리 → 사실상 연결 리스트처럼 생김
- 트리의 높이 h가 노드 수 n과 거의 같아짐
- 탐색/삽입/삭제가 root→leaf 까지 일직선으로 ⇒ O(h) = O(n) 시간

## AVL 트리는 어떻게 균형을 유지하는가? 회전의 종류는?

- AVL 트리
    - 각 노드에서 L/R 서브트리 높이 차이가 최대 1만 허용하는 균형 BST
- 삽입/삭제 후에 어떤 노드의 균형이 깨지면(높이 차이 2 이상) 회전으로 다시 맞춤
- 회전 종류
    1. LL 회전 (왼쪽-왼쪽 불균형 → 오른쪽으로 회전)
    2. RR 회전 (오른쪽-오른쪽 → 왼쪽으로 회전)
    3. LR 회전 (왼쪽-오른쪽 → 왼쪽 회전 후 오른쪽 회전)
    4. RL 회전 (오른쪽-왼쪽 → 오른쪽 회전 후 왼쪽 회전)

## Red-Black 트리의 특성은 무엇인가?

- 균형 BST
- 규칙을 느슨하게 가져가는 대신 연산이 더 가벼움
- 대표 규칙
    - 각 노드는 빨강 또는 검정
    - 루트는 항상 검정
    - 모든 리프(NIL) 노드는 검정
    - 빨강 노드의 자식은 둘 다 검정 (빨강이 연속해서 나오지 않음)
    - 루트에서 임의의 리프(NIL)까지 가는 모든 경로에서 검정 노드 개수는 같음

이 규칙 덕분에 트리 높이가 항상 O(log n)에 bounded 됨 → 탐색/삽입/삭제 모두 O(log n) 보장

## AVL 트리와 Red-Black 트리 중 어떤 것을 선택해야 하는가?

- AVL 트리
    - 더 강하게 균형을 맞춤 → 탐색 속도가 약간 더 좋음
    - 삽입/삭제 시 회전이 더 자주 발생해 구현이 조금 복잡하고 업데이트 비용이 큼
    
    ⇒ 읽기(검색) 위주인 경우에 유리
    
- Red-Black 트리
    - 균형 조건이 좀 더 느슨함 → 트리 높이가 AVL보다 약간 더 클 수 있음
    - 삽입/삭제 시 회전이 덜 필요해서 업데이트가 더 가벼움
    
    ⇒ 라이브러리, 언어 표준 컨테이너 등 실무에서 많이 쓰는 쪽 (C++ map/set 등)에 유리
    

즉, 탐색 성능을 극대화하고 싶으면 AVL, 삽입/삭제가 잦고 전체적인 성능 및 구현 단순성을 고려하면 Red-Black
