### 📌 [Q1] 정규화가 필요한 이유는? 이상 현상(Anomaly)이란?

**정규화(Normalization)**
- 데이터베이스의 데이터를 효율적으로 관리하고, 데이터의 일관성과 정확성을 유지하기 위한 프로세스
- 데이터의 중복을 최소화하고, 데이터의 일관성을 유지하며, 데이터의 정확성을 보장
- **함수적 종속성**을 기반으로 테이블을 분해해 중복 제거 & 이상 현상 방지

**이상 현상(Anomaly)**
- 데이터베이스에서 발생하는 데이터의 일관성과 정확성을 해치는 현상
- 테이블 설계가 잘못되어 데이터의 삽입, 수정, 삭제 시 발생하는 논리적 오류나 데이터 불일치 현상

### 📌 [Q2] 제1정규형, 제2정규형, 제3정규형, BCNF의 조건과 차이는?

데이터베이스 정규화는 단계가 높아질수록 더 엄격한 제약 조건을 적용해 데이터 중복을 제거하며, 각 단계는 이전 단계의 조건을 모두 만족해야 한다.

**제1정규형(1NF)**
- 모든 도메인이 원자 값(atomic value)으로 구성되어야 함
- 한 칸에는 하나의 값만 들어가야 함
- 중복 값, 반복 그룹 제거

**제2정규형(2NF)**
- 부분 함수 종속성 제거
- 기본키의 일부분에만 종속되는 속성 분리

**제3정규형(3NF)**
- 이행적 함수 종속성 제거
- 기반키가 아닌 속상 간의 종속 관계 분리

**BCNF**
- 모든 결정자가 후보키여야 함
- 후보키가 여러 개인 경우 발생하는 종속성 해결

### 📌 [Q3] 함수적 종속성(Functional Dependency)이란 무엇인가?

**함수적 종속성(Functional Dependency)**
- X(결정자) -> Y(종속 속성)
- X의 값이 Y의 값에 영향을 미치는 관계
- X가 Y를 함수적으로 결정한다 (X의 값이 같으면 Y의 값도 반드시 같음)

ex)
```
학번 → 이름
학번 → 학과
```
- 같은 학번을 가진 학생은 항상 같은 이름과 학과를 가짐
- 학번 하나로 학생 정보가 결정됨

### 📌 [Q4] 정규화의 장단점은? 왜 항상 정규화를 하지 않는가?

**장점**
- 데이터 중복 최소화
- 이상 현상(Anomaly) 방지
- 데이터 무결성 향상 (한 정보가 한 곳에만 존재)
- 데이터베이스 성능 향상

**단점**
- 데이터베이스 설계 복잡도 증가 (쿼리 복잡도 증가)
- 데이터베이스 관리 복잡도 증가
- 데이터베이스 성능 저하 가능성 (JOIN 증가 -> 조회 성능 저하, 테이블을 여러개로 나누기 때문, 정규화는 읽기 성능보다 쓰기 무결성에 초점)

**항상 정규화를 하지 않는 이유**
- 성능 저하 가능성 때문
  - 실제 서비스에서는 조회(read)가 압도적으로 많지만, 정규화는 읽기 성능보다 쓰기 무결성에 초점을 둠
  - 이렇게 되면 트래픽이 많은 서비스 / 조회 중심 시스템에서는 성능이 저하됨 -> 이때 일부러 반/역정규화(Denormalization)를 하기도 함

### 📌 [Q5] 역정규화는 언제 하는가? 어떤 경우에 성능 향상이 되는가?

**역정규화(Denormalization)**
- 정규화를 통해 나눈 테이블을 의도적으로 다시 합치거나 중복 칼럼을 추가하는 것
- 데이터베이스의 무결성을 유지하면서도 성능 향상을 목적으로 데이터를 복제하는 방법
- 목적: 조회(read) 성능 향상

**역정규화의 경우**
- 조회가 압도적으로 많은 시스템
  - 커뮤니티, 쇼핑몰 상품 리스트, sns 피드, 통계 대시보드 등 쓰기보다 읽기가 훨씬 많은 상황
- JOIN이 너무 많아서 병목이 발생할 때
  - JOIN이 많을 수록 (디스크 I/O 증가, 정렬/해시 비용 증가, 실행 계획 복잡도 증가) -> 응답 속도 느려짐 -> 일부 데이터를 중복 저장


### 📌 [Q6] 실무에서는 보통 몇 정규형까지 정규화하는가?

실무에서는 보통 **3정규형**까지 적용하는 경우가 가장 많다.

- 1NF — 원자값
- 2NF — 부분 함수 종속 제거
- 3NF — 이행적 종속 제거
- BCNF
- 4NF, 5NF ...

3NF까지 정규화를 하면
- 데이터 중복 대부분 제거
- 삽입/삭제/갱신 이상 거의 해결
- 현실적인 데이터 무결성 확보
- JOIN 복잡도 과도하게 증가하지 않음
즉! 무결성과 성능의 균형점이 바로 3NF
