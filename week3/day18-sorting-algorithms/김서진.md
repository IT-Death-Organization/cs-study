## 1.  병합 정렬의 분할 정복 원리를 설명하면?
    병합 정렬은 문제를 작게 나누고 -> 각각 해결한 뒤 -> 다시 합치는 방식으로 정렬을 수행한다. 
        1) Divide: 배열을 절반으로 계속 나눈다
        2) Conquer: 더 이상 나눌 수 없는 크기(원소 1)가 되면 정렬된 상태로 본다.
        3) Combine: 두 개의 정렬된 배열을 비교하면서 하나의 정렬된 배열로 합친다.
    예) [38, 27, 43, 3, 9, 82, 10]
        1) Divide: 
            [38, 27, 43, 3] [9, 82, 10]
            [38, 27] [43, 3] [9, 82] [10]
        2) Conquer:
            [38] [27] [43] [3] [9] [82] [10]
        3) Combine:
            [27, 38] [3, 43] [9, 82] [10]
            [3, 27, 38, 43] [9, 10, 82]
            [3, 9, 10, 27, 38, 43, 82]
    병합 정렬의 특징으로는, 분할 단계에서는 비교 연산이 거의 없으며, 병합 단계에서 정렬이 이루어진다. 또한 항상 반으로 나누는 구조이다.

---

## 2.  병합 정렬이 항상 O(n log n)인 이유는?
    병합 정렬의 시간 복잡도는 두가지로 결정된다.
    1) 분할 깊이 = Log n
        배열을 계속 절반으로 나누기 때문! 
        n -> n/2 -> n/4 -> ... -> 1
        2^k = n
        k = log n
    
    2) 각 단계에서 수행하는 작업 = n
        각 단계에서 병합 작업이 수행된다. n개의 원소를 하나씩!
    
    => 전체 시간 = n * log n

    병합 정렬이 항상 O(n log n)인 이유는 단순하다! 데이터 상태에 영향을 받지 않기 때문이다. 그리고 항상 정확히 절반씩 분할하며 병합 과정도 항상 n번 처리하기 때문이다.

---

## 3.  병합 정렬의 공간복잡도가 O(n)인 이유는?
    병합 정렬은 병합 과정에서 새로운 배열(임시 공간)을 사용하기 때문이다.
    예를 들어, [1,2,3]과 [4,5,6]이라는 두가지 배열을 비교할때, 새로운 배열에 정렬된 원소를 넣는다. 즉 temp: [1,2,3,4,5,6]이라는 임시 배열이 생성된다. n개의 원소를 가진 배열이라면, temp 배열을 n번 생성할 것이다. 따라서 추가 메모리는 O(n)이다.

    또한, 병합 정렬은 재귀로 구현된다. 재귀 깊이는 log n이다. 따라서 재귀 스택도 O(log n)일것이다. 

    전체 공간복잡도는 O(n)과 O(log n) 중 더 큰 O(n)이 최종 공간봅잡도가 된다.

---

## 4.  퀵 정렬이 평균적으로 병합 정렬보다 빠른 이유는?
    퀵 정렬은 배열에서 기준값(pivot)을 하나 선택하고, 그 값을 기준으로 작은 값과 큰 값을 나눠 정렬한다.
        1) pivot 선택
        2) pivot 기준으로 작은 값 / 큰 값으로 분할 (partition)
        3) pivot 기준으로 좌우 배열 분할
        4) 좌측 quickSort
        5) 우측 quickSort

    예) [8, 3, 1, 7, 0, 10, 2]
        1) 피봇 선택: pivot = 8
            보통 첫 번째 값 / 마지막 값 / 가운데 값 등을 사용한다.
        2) partition(분할): [3, 1, 7, 0, 2 | 8 | 10]
            pivot보다 작은 값은 왼쪽, 큰 값은 오른쪽으로 이동
        3) 재귀 정렬
            왼쪽 배열: [3,1,7,0,2]
            오른쪽 배열: [10]
            최종 결과 => [0,1,2,3,7,8,10]

    이론적으로 퀵 정렬과 병합 정렬 모두 O(n log n)이다. 하지만 실제 실행 속도는 퀵 정렬이 더 빠른 경우가 많다. 
        1) 추가 메모리를 사용하지 않는다. 퀵 정렬은 배열 내부에서 정렬한다.
        2) 캐시 효율이 좋다. 배열 내부에서 swap을 반복하기 때문이다.
        3) 상수 계수가 작다. quick sort는 비교 + swap이라는 빠른 알고리즘이다. 

---

## 5.  퀵 정렬의 최악의 경우는 언제 발생하며 시간복잡도는?
    퀵 정렬의 최악의 경우는 피벗이 항상 가장 작은 값 또는 가장 큰 값으로 선택되는 경우에 발생한다. 이 경우, 배열이 매번 한쪽으로만 분할된다. 예를 들어, [1,2,3,4,5]인 배열에서 피벗이 1이 되는 경우, O(n^2)의 시간복잡도를 가지게 된다.
    대표적인 발생 상황으로는 이미 정렬된 배열이거나, 역순 배열, pivot을 항상 첫 원소 또는 마지막 원소로 선택하는 경우에 발생하게 된다.

---

## 6.  피벗 선택 방법(첫 번째, 마지막, 중간, 랜덤)에 따른 성능 차이는?
    퀵 정렬의 성능은 pivot 선택 전략에 크게 영향을 받는다.
    1) 첫번째 원소
        - 구현이 매우 단순하다는 특징이 있다. 
        - 그러나 정렬된 배열에서는 최악이 발생된다.
        - 평균 O(n log n)
        - 최악 O(n^2)
    
    2) 마지막 원소
        - parition 구현이 쉽다.
        - 이 또한 이미 정렬된 데이터에서 최악이 발생한다.
        - 평균 O(n log n)
        - 최악 O(n^2)
    
    3) 중간 원소
        - 정렬된 배열에서도 균형 분할 가능성이 높음
        - 첫 원소 pivot보다 안정적임!
    
    4) 랜덤 원소
        - 최악의 경우 발생 확률이 매우 낮음~
        - 기대 시간복잡도는 O(n log n)

---

## 7.  힙 정렬의 시간복잡도와 공간복잡도는? 왜 O(n log n)인가?
    힙 정렬은 힙 자료구조를 이용한 정렬이다. 보통 최대 힙을 만들어 가장 큰 값을 하나씩 꺼내며 정렬한다.
        1) 배열을 최대 힙으로 변환 (Heapify)
        2) 루트(최댓값)를 배열 끝과 교환
        3) 힙 크기 감소
        4) 다시 heapify 수행
        5) 위 과정 반복
    
    <시간복잡도>
    힙 정렬은 두 단계로 구성된다.
        1) 힙 생성: 배열을 힙 구조로 만드는 과정 => O(n)
        2) 정렬 과정: 루트 노드를 하나씩 꺼낸다 => O(n * log n)
    => 시간복잡도: O(n log n)

    <공간복잡도>
    힙 정렬은 추가 배열을 사용하지 않는다.
    즉, O(1)이다!

## 8.  실무에서 어떤 정렬 알고리즘을 사용하는가? (Tim Sort, Intro Sort)
    실무에서는 순수 퀵정렬, 병합정렬, 힙정렬을 그대로 사용하지 않고, 하이브리드 정렬 알고리즘을 쓴다. 대표적으로 Tim sort와 Intro sort가 있다.

    1) Tim Sort
    Tim Sort는 Merge Sort + Insertion Sort를 결합한 알고리즘이다.

    이미 정렬된 구간(run)을 탐지
        → 작은 구간은 insertion sort
        → 큰 구간은 merge sort
    
    장점으로는 
    - 부분 정렬된 데이터에 매우 빠름
    - 안정 정렬
    - 실제 데이터에 매우 효율적

    시간복잡도는 
    Best   O(n)
    Avg    O(n log n)
    Worst  O(n log n)


    2) Intro Sort (Introspective Sort)
    Intro Sort는 Quick Sort + Heap Sort + Insertion Sort의 결합이다.

    Quick Sort로 시작
        → 재귀 깊이가 너무 깊어지면
        → Heap Sort로 전환

    Best   O(n log n)
    Avg    O(n log n)
    Worst  O(n log n)
