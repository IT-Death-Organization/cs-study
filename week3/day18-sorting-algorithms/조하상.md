## 1. 병합 정렬의 분할 정복 원리를 설명하면?

**단계별 처리 과정**
- **분할(Divide):** 정렬되지 않은 리스트를 절반으로 나누어 두 개의 하위 리스트로 분리한다. 이 과정을 리스트의 크기가 1이 될 때까지 재귀적으로 반복한다.
- **정복(Conquer):** 각 하위 리스트는 크기가 1일 때 이미 정렬된 것으로 간주한다.
- **결합(Combine):** 정렬된 하위 리스트들을 하나의 정렬된 리스트로 합친다. 두 리스트의 요소를 비교하며 작은 순서대로 새 리스트에 옮겨 담는다.



---

## 2. 병합 정렬이 항상 O(n log n)인 이유는?

**일정한 분할과 비교 횟수**
- 데이터의 분포와 상관없이 항상 리스트를 절반으로 나누기 때문에 트리의 높이가 항상 $\log n$이 된다.
- 각 층(Level)에서 병합 연산을 수행할 때 모든 요소($n$개)를 한 번씩 비교하므로, 매 층마다 $O(n)$의 시간이 소요된다.
- 따라서 전체 시간 복잡도는 $n \times \log n$, 즉 항상 $O(n \log n)$을 유지한다.

---

## 3. 병합 정렬의 공간복잡도가 O(n)인 이유는?

**추가 메모리 공간 할당**
- 병합 정렬은 두 개의 하위 리스트를 합쳐서 정렬된 결과를 만들 때, 이를 임시로 저장할 **별도의 배열(Additional Array)**이 반드시 필요하다.
- 입력 배열의 크기인 $n$만큼의 추가 공간을 매번 혹은 최종적으로 할당해야 하므로 공간 복잡도는 $O(n)$이 된다.

---

## 4. 퀵 정렬이 평균적으로 병합 정렬보다 빠른 이유는?

**제자리 정렬과 참조 지역성**
- 퀵 정렬은 별도의 추가 메모리 공간을 생성하지 않고 기존 배열 내에서 위치를 바꾸는 **제자리 정렬(In-place Sort)** 방식을 사용한다.
- 데이터 이동이 상대적으로 적고, CPU 캐시 메모리를 효율적으로 활용할 수 있는 **참조 지역성(Locality of Reference)**이 뛰어나기 때문에 실제 실행 속도에서 병합 정렬보다 우위를 점한다.



---

## 5. 퀵 정렬의 최악의 경우는 언제 발생하며 시간복잡도는?

**불균형한 분할의 발생**
- **발생 상황:** 피벗(Pivot)이 매번 부분 배열 내에서 최솟값이나 최댓값으로 선택될 때 발생한다. 주로 이미 정렬된 배열이나 역순 배열에서 첫 번째 요소를 피벗으로 잡을 때 나타난다.
- **시간 복잡도:** 분할이 $n$번 일어나고 각 단계에서 $O(n)$의 비교를 수행하게 되어 $O(n^2)$이 된다.

---

## 6. 피벗 선택 방법(첫 번째, 마지막, 중간, 랜덤)에 따른 성능 차이는?

- **첫 번째/마지막 요소:** 구현이 가장 간단하지만 정렬된 데이터에 매우 취약하여 최악의 성능($O(n^2)$)을 낼 확률이 높다.
- **중간 요소 (Median):** 이미 정렬된 배열에서도 비교적 균등하게 분할될 확률을 높여주어 성능이 안정적이다.
- **랜덤 요소:** 피벗을 무작위로 선택하여 최악의 케이스를 만날 확률을 수학적으로 극히 낮춘다.
- **Median-of-Three:** 첫 번째, 중간, 마지막 값 중 중간값을 선택하는 방식으로, 실무에서 퀵 정렬의 성능을 보장하기 위해 가장 많이 사용한다.

---

## 7. 힙 정렬의 시간복잡도와 공간복잡도는? 왜 O(n log n)인가?

- **시간 복잡도:** $O(n \log n)$이다. 초기 힙 구성(Heapify)에 $O(n)$이 걸리고, 루트 노드를 추출하고 다시 힙을 재구조화하는 과정을 $n$번 반복하는데 각 재구조화에 $O(\log n)$이 걸리기 때문이다.
- **공간 복잡도:** 추가 배열 없이 기존 배열을 힙 구조로 변환하여 정렬하는 제자리 정렬 방식이므로 $O(1)$이다.



---

## 8. 실무에서 어떤 정렬 알고리즘을 사용하는가? (Tim Sort, Intro Sort)

**하이브리드 알고리즘의 채택**
- **Tim Sort:** **삽입 정렬과 병합 정렬**을 결합한 방식이다. 현실 데이터가 어느 정도 정렬되어 있다는 가정을 활용하여 최선의 경우 $O(n)$, 최악의 경우 $O(n \log n)$을 보장하며 Java, Python, JavaScript 등의 표준 정렬로 쓰인다.
- **Intro Sort:** **퀵 정렬**로 시작하여 재귀 깊이가 일정 수준 이상 깊어지면 **힙 정렬**로 전환하고, 데이터 개수가 적으면 **삽입 정렬**을 수행한다. 최악의 경우에도 $O(n \log n)$을 보장하며 C++의 `std::sort` 등에서 사용된다.
