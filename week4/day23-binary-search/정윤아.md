## Q1. 이진 탐색의 전제 조건은 무엇이며 왜 필요한가?

[답변]
* 전제 조건: 탐색 대상이 정렬되어 있다
  - 정렬되어 있어야 중간값을 기준으로 찾는 값이 위치한 구간이 왼쪽인지 오른쪽인지 판단 가능
  - 판단 기준이 단조성 또는 순서성을 가져야 함

---

## Q2. 이진 탐색이 O(log n)인 이유를 수학적으로 설명하면?

[답변]
* 매 탐색마다 탐색 범위 절반으로 줄임
  - 처음 원소가 n개
  - 1번 비교 후 남는 범위: n / 2
  - 2번 비교 후 남는 범위: n / 4
  - 3번 비교 후 남는 범위: n / 8
  - ...
  - k번 비교 후 남는 범위: n / 2^k
  - 탐색 종료: n / 2^k <= 1
    => 2^k >= n
    => k >= log2(n) : k(실행 횟수)는 약 log2(n)번
    => O(logn)

## Q3. 이진 탐색 구현 시 mid 계산을 (low + high) / 2 대신 low + (high - low) / 2로 하는 이유는?

[답변]
* mid = (low + high) / 2
  - 정수 오버플로우 문제 발생 가능
  - low, high가 매우 큰 정수라면, 두 값을 더하는 순간 자료형 범위를 초과할 수 있음
* mid = low + (high - low) / 2
  - high - low 먼저 계산 => 불필요하게 큰 수의 덧셈 피함
  - 더 안전한 구현 방식

## Q4. Lower Bound와 Upper Bound의 차이는? 각각 언제 사용하는가?

[답변]
* Lower Bound
  - target 이상(>= target)이 처음 나타나는 위치
* Upper Bound
  - target 초과(> target)가 처음 나타나는 위치
 
ex. [1, 2, 2, 2, 4, 5], target=2
  - Lower Bound = 인덱스 1 (값: 2)
  - Upper Bound = 인덱스 4 (값: 4)

* 사용 경우
  - 중복 원소가 있는 정렬 배열에서 중요
  - 값 위치 찾기
    - Lower Bound: 어떤 값이 처음 등장하는 위치 찾기
    - Upper Bound: 어떤 값이 마지막으로 등장한 다음 위치 찾기
  - 특정 값 개수 구하기: upper_bound - lower_bound
  - 삽입 위치 찾기
    - Lower Bound: 정렬 순서를 유지하면서 target을 삽입할 수 있는 가장 앞 위치
    - Upper Bound: 같은 값들 뒤에 삽입할 수 있는 위치
  - 구현 관점
    - Lower Bound: arr[mid] >= target
    - Upper Bound: arr[mid] > target

## Q5. 이진 탐색을 응용할 수 있는 문제 유형은? (parametric search)

[답변]
* 값 찾기
* 정답 구간 탐색: 파라메트릭 서치(parametric search)
  - 어떤 값 x가 조건을 만족하는지 검사하는 함수가 있음
    (ex. 길이 x로 자를 수 있는가, 시간 x 안에 끝낼 수 있는가 등 -> 정답 후보 값 x 자체를 놓고 가능/불가능 판별)
  - 이 함수의 결과가 특정 구간부터 단조적으로 변화시 (결과가 한 번만 바뀜)
  - 가능한 정답 범위를 이진 탐색으로 줄일 수 있음

ex.
* 문제: 길이가 서로 다른 랜선들이 있을 때, 잘라서 최소 6개 이상 만들고 싶다
* 랜선 길이 배열: [8, 5, 7]

“자를 수 있는 최대 길이”를 구하는 문제

정답을 직접 구하지 않고, 길이 x를 정해서 질문
  => 길이 x로 잘랐을 때 6개 이상 만들 수 있는가?

* x = 1
  - 8//1 + 5//1 + 7//1 = 20
	- 가능
* x = 2
  - 8//2 + 5//2 + 7//2 = 4 + 2 + 3 = 9
	- 가능
* x = 3
  - 8//3 + 5//3 + 7//3 = 2 + 1 + 2 = 5
	- 불가능
* x = 4
  - 8//4 + 5//4 + 7//4 = 2 + 1 + 1 = 4
	- 불가능

즉 결과는
  x = 1 → 가능
	x = 2 → 가능
	x = 3 → 불가능
	x = 4 → 불가능

=> 가능한 최대 길이는 2
  - 해당 경계를 이진 탐색으로 찾을 수 있음
  - x = 1, 2, 3, 4 순서대로가 아닌 2나 3부터 잘라봄

## Q6. 회전된 정렬 배열에서 이진 탐색을 어떻게 적용하는가?

[답변]
* 회전된 정렬 배열
  - 정렬된 배열을 중간에서 끊어서 앞뒤를 바꾼 배열
    ex. [1, 2, 3, 4, 5, 6, 7]을 4 앞에서 끊어 뒤로 보냄
    =>  [4, 5, 6, 7, 1, 2, 3] : '4칸 회전'
  - mid를 기준으로 한쪽 구간은 반드시 정렬
    - mid를 구함
    - arr[low] ~ arr[mid]가 정렬되어 있는지 확인
      - 정렬 O: target이 그 구간 안에 포함되는지 검사
        - 포함 O: high = mid-1
        - 포함 X: low = mid+1
      - 정렬 X: 오른쪽 arr[mid] ~ arr[high]는 정렬됨
        - target이 오른쪽 정렬 구간 안에 있는지 검사해 범위를 줄임
       
    => 어느 쪽이 정렬된 구간인지 파악 후, 그 안에 target이 있는지 확인하는 방식

[보완]
* 정렬 판단
  - arr[low] <= arr[mid] 이면 왼쪽 절반이 정렬
  - 그렇지 않으면 오른쪽 절반이 정렬
* 왼쪽 정렬 시
  - arr[low] <= target < arr[mid] 이면 왼쪽으로 이동
  - 아니면 오른쪽으로 이동
  - O(logn)
  - 중복 원소가 많은 경우, 최악의 경우 O(n)
 
* 정렬되지 않은 쪽에 포함된 경우, 정렬되지 않은 쪽을 먼저 봐선 판단 불가 -> 정렬된 쪽을 먼저 봐서 target이 포함되는지 안 되는지 판단
