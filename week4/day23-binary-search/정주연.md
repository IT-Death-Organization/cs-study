# Day 23: 이진 탐색(Binary Search)

## 🧠 학습 질문

## 이진 탐색의 전제 조건은 무엇이며 왜 필요한가?

- **전제 조건**
    - 데이터가 반드시 정렬되어 있어야 함
- **전제 조건의 필요성**
    - 이진 탐색은 중앙값과 타겟을 비교하여 탐색 범위를 절반으로 줄여나가는 방식
    - 데이터가 정렬되어 있지 않다면 중앙값보다 큰 데이터가 왼쪽에 있을지 오른쪽에 있을지 판단할 수 없으므로 이 알고리즘의 논리 자체가 성립하지 않음

## 이진 탐색이 O(log n)인 이유를 수학적으로 설명하면?

- 데이터의 개수를 $n$이라 할 때, 한 번의 비교를 거칠 때마다 탐색 범위는 1/2 씩 줄어듦
- 1회 수행 후 남은 데이터: n * (1/2)
- 2회 수행 후 남은 데이터: n * (1/2)^2
- 탐색이 종료되는 시점은 남은 데이터가 1개일 때 →  n* (1/2) ^k = 1
- 양변에 2^k 곱 → k  = logn 됨

## 이진 탐색 구현 시 mid 계산을 (low + high) / 2 대신 low + (high - low) / 2로 하는 이유는?

- 오버플로우 방지
    - low와 high가 매우 큰 값일 때, 둘을 더하는 과정에서 정수 오버플로우(Integer Overflow)가 발생하여 잘못된 음수값이 나올 수 있음
    - 개선된 식은 high - low를 먼저 계산하므로 오버플로우 위험 없이 안전하게 중앙값을 구할 수 있음

## Lower Bound와 Upper Bound의 차이는? 각각 언제 사용하는가?

- 정렬된 배열에서 특정 값이 중복될 때, 그 구간을 찾기 위해 사용함

| **구분** | **정의** | **사용 시점** |
| --- | --- | --- |
| **Lower Bound** | 찾고자 하는 값 **이상의 값**이 처음 나타나는 위치 | 특정 값의 시작 인덱스 찾기 |
| **Upper Bound** | 찾고자 하는 값을 **초과하는 값**이 처음 나타나는 위치 | 특정 값의 끝 다음 인덱스 찾기 |

## 이진 탐색을 응용할 수 있는 문제 유형은? (parametric search)

- parametric search
    - 최적화 문제(찾고자 하는 값의 최댓값/최솟값 구하기)를 결정 문제(조건을 만족하는가? Yes/No)로 바꾸어 이진 탐색으로 해결하는 기법
- 사용 조건
    - 상태 함수의 결과가 단조성(Monotonicity)을 띄어야 함 (e.g.  x에서 조건을 만족하면 x-1에서도 반드시 만족해야 함)
- 문제 유형
    - 정답의 범위 내에서 이진 탐색을 수행함
    - e.g. 나무를 M미터 얻기 위해 절단기 높이를 얼마로 설정해야 하는가?
    - e.g. 예산 범위 내에서 가능한 최대 금액은?

## 회전된 정렬 배열에서 이진 탐색을 어떻게 적용하는가?

정렬된 배열이 특정 축을 기준으로 회전된 상태(예: [4, 5, 6, 7, 0, 1, 2])에서도 이진 탐색을 적용할 수 있음

- 배열을 절반으로 나누면, **최소한 한쪽 부분은 반드시 정렬된 상태**임
- 핵심 로직
    - mid를 기준으로 왼쪽(low ~ mid)이 정렬되어 있는지 확인
    - 왼쪽이 정렬되어 있다면, 타겟이 그 범위 안에 있는지 체크하고 탐색 범위를 좁힘
    - 왼쪽이 정렬되어 있지 않다면, 오른쪽(mid ~ high)이 반드시 정렬되어 있으므로 같은 방식으로 타겟을 체크
- 이 방식을 통해 회전된 배열에서도 O(log n)의 성능을 유지할 수 있음
