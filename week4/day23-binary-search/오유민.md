### 📌 [Q1] 이진 탐색의 전제 조건은 무엇이며 왜 필요한가?

**전제 조건**
- **정렬된 배열**이어야 한다.
  - 데이터가 오름차순 또는 내림차순으로 미리 정렬되어 있어야 한다.
  - **임의 접근(Random Access)**이 가능한 데이터 구조(배열 등)여야 $O(1)$로 중간값에 접근할 수 있어 성능이 보장된다.

**필요성**
- 이진 탐색은 **분할 정복(Divide and Conquer)** 원리를 사용하여 탐색 범위를 절반씩 줄여나가는데, 이때 **정렬 상태**가 판단의 근거가 되기 때문이다.
- 중간 요소(mid)와 찾는 값(target)을 비교했을 때, target < mid이면 "데이터가 정렬되어 있으므로" 오른쪽 절반은 확인할 필요가 없다는 확신을 가질 수 있다.
- 정렬되어 있지 않다면 어느 쪽 절반에 값이 있을지 알 수 없으므로, 결국 모든 데이터를 확인하는 선형 탐색($O(N)$)을 수행해야 한다.

### 📌 [Q2] 이진 탐색이 O(log n)인 이유를 수학적으로 설명하면?

**수학적 명해**
1. 처음 전체 데이터 개수를 $N$이라 할 때, 1회 탐색 시마다 남는 데이터는 $\frac{1}{2}$씩 줄어든다.
2. $k$번 탐색 후 남은 데이터의 개수는 $N \times (\frac{1}{2})^k$가 된다.
3. 최악의 경우 데이터가 1개가 남을 때까지 탐색하므로, $N \times (\frac{1}{2})^k = 1$이 되는 시점이 종료 시점이다.
4. 위 식을 정리하면 $N = 2^k$가 되며, 양변에 로그를 취하면 $k = \log_2 N$이 된다.
- 따라서 탐색 횟수 $k$는 데이터 크기 $N$에 로그 비례하므로 시간 복잡도는 **$O(\log N)$**이다.

### 📌 [Q3] 이진 탐색 구현 시 mid 계산을 (low + high) / 2 대신 low + (high - low) / 2로 하는 이유는?

**이유**
- **정수 오버플로우(Integer Overflow)**를 방지하기 위해서이다.
- **(low + high) / 2의 문제점**: `low`와 `high`가 모두 큰 값인 경우, 두 값의 합이 해당 자료형(int 등)이 표현할 수 있는 최대 범위를 초과하여 잘못된 결과(음수 등)를 반환할 수 있다.
- **low + (high - low) / 2가 안전한 이유**: 두 값의 '차이'를 먼저 구한 뒤 `low`에 더해주므로, 중간 계산 과정에서 절대 `high` 값을 넘지 않아 오버플로우로부터 안전하다.
- **기타 방법**: 자바 등 일부 언어에서는 `(low + high) >>> 1` (Unsigned Right Shift) 연산을 사용하여 오버플로우 문제를 해결하기도 한다.

### 📌 [Q4] Lower Bound와 Upper Bound의 차이는? 각각 언제 사용하는가?

**정의 및 차이**
- **Lower Bound (하한)**: 원하는 값(target) **이상**의 값이 처음으로 나타나는 위치를 반환한다.
- **Upper Bound (상한)**: 원하는 값(target)을 **초과**하는 값이 처음으로 나타나는 위치를 반환한다.

**사용 사례**
1. **중복된 원소의 개수 구할 때**: `Upper Bound - Lower Bound`를 통해 특정 값의 빈도수를 $O(\log N)$으로 구할 수 있다.
2. **범위 쿼리**: 특정 범위 $[A, B]$ 내에 속하는 숫자의 개수를 셀 때 유용하다.
3. **삽입 위치 탐색**: 정렬 순서를 유지하면서 새 데이터를 삽입할 수 있는 가장 앞 혹은 가장 뒤의 위치를 찾을 때 사용한다.

### 📌 [Q5] 이진 탐색을 응용할 수 있는 문제 유형은? (parametric search)

**핵심 기법: 파라메트릭 서치 (Parametric Search)**
- 최적화 문제(최대값/최소값 찾기)를 결정 문제(Yes/No)로 바꾸어 푸는 기법이다.
- "조건을 만족하는 가장 큰/작은 $X$는 무엇인가?" $\rightarrow$ "특정 값 $K$일 때 조건을 만족하는가?" ($O(1)$ 또는 $O(N)$으로 판별)

**대표 문제 유형**
- **자르기 문제**: 나무 자르기, 랜선 자르기 등 특정 길이를 기준으로 조건을 만족하는 최대 길이를 구하는 경우.
- **최소값의 최대화 / 최대값의 최소화**: 공유기 설치(두 공유기 사이의 최소 거리를 최대화)와 같은 배치 문제.
- **데이터 범위가 매우 큰 경우**: 정답의 범위가 $10^9$ 이상일 때 선형 탐색 대신 이진 탐색으로 효율적인 탐색이 가능하다.

### 📌 [Q6] 회전된 정렬 배열에서 이진 탐색을 어떻게 적용하는가?

**핵심 아이디어: "반드시 한쪽은 정렬되어 있다"**
- 회전된 정렬 배열(예: `[4, 5, 6, 7, 0, 1, 2]`)을 `mid`로 나누면, 왼쪽 반 또는 오른쪽 반 중 하나는 반드시 정렬된 상태를 유지한다.

**탐색 방법**
1. `mid`를 기준으로 `arr[low] <= arr[mid]`인지 확인하여 **왼쪽이 정렬되었는지** 판단한다.
2. **왼쪽이 정렬된 경우**: `target`이 `arr[low]`와 `arr[mid]` 사이에 있다면 왼쪽을 탐색, 아니면 오른쪽을 탐색한다.
3. **오른쪽이 정렬된 경우(반대)**: `target`이 `arr[mid]`와 `arr[high]` 사이에 있다면 오른쪽을 탐색, 아니면 왼쪽을 탐색한다.
- 이 원리를 이용하면 회전된 배열에서도 여전히 **$O(\log N)$** 속도로 타겟을 찾을 수 있다.
