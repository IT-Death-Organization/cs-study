## 1. 이진 탐색의 전제 조건은 무엇이며 왜 필요한가 ?
> 이진 탐색은 **정렬된 배열**에서 특정 값을 효율적으로 찾기 위한 알고리즘입니다.
- 이진 탐색은 중간 지점을 기준으로 탐색 범위를 절반씩 줄여나가는 **분할 정복** 방식을 사용합니다.
- 이진 탐색의 전제 조건은 **정렬된 배열**이어야 하며, 이는 절반씩 줄여나가는 방식으로 인해 효율적인 검색을 가능케 합니다.
### 1-1. 배열이 정렬돼 있어야 하는 이유
- 정렬되지 않은 배열에서는 이진 탐색을 적용할 수 없기 때문에, 정렬된 배열에서만 사용할 수 있는 중요한 조건입니다.

→ 정렬된 배열에서 특정 값을 찾는 알고리즘으로, 이진 탐색은 시간 복잡도 **O(log n)** 를 가지며, 대규모 데이터셋에서도 빠르게 원하는 값을 찾을 수 있습니다.

## 2. 이진 탐색이 O(log n)인 이유를 수학적으로 설명하면?
- 이진 탐색은 데이터가 정렬돼 있어야 하며, 매번 중간값을 기준으로 탐색을 반복할 때마다 탐색 범위를 반으로 줄여 나가는 방식으로 작동합니다.
- 이러한 탐색 범위가 더 이상 줄일 수 없는 1일 될 때의 탐색 횟수를 K라고 한다면, 아래 표와 같습니다.

| 비교 | 범위 | 
| --- | --- | 
| 0 | n |
| 1 | n/2 | 
| 2 | n/4 |
| ... | ... |
| k | n/2^k |

▶ 표의 마지막 행에서 n/2^k = 1이므로, k = log₂ n 임을 알 수 있습니다.

∴ **이진 탐색의 시간 복잡도는 O(log n)** 이며, 최악의 경우에도 log₂ n번만 비교하면 원하는 데이터를 찾을 수 있습니다.

## 3. 이진 탐색 구현 시 mid 계산을 (low + high) / 2 대신 low + (high - low) / 2로 하는 이유는?
> 이진 탐색 구현 시 mid 계산을 low + (high - low) /2 로 하는 이유, **배열의 길이가 2^n 인 경우에 대해 효율적인 탐색을 수행하기 위함입니다.** 이 방법은 배열의 시작 인덱스와 끝 인덱스를 고려해 mid를 계산하며, 이로 인해 배열의 길이가 2^n 인 경우에도 불구하고 O(log n)의 시간 복잡도를 유지할 수 있습니다.
### 3-1. 정수 오버플로(Integer Overflow) 방지
대부분의 프로그래밍 언어에서 정수형(Int) 은 메모리 크기에 따라 표현할 수 있는 **최대 범위** 가 정해져 있습니다.
### 기존 방식 : `(low + high) / 2`
- 문제점 : `low` 와 `high` 가 각각은 정수 범위 내에 있더라도, 둘을 합친 `low + high` 가 해당 자료형의 최대치를 넘어설 수 있습니다.
- 결과 : 오버플로가 발생하면 합산 결과가 **음수** 로 변하거나 엉뚱한 값으로 튀게 되어, 인덱스 참조 시 에러가 발생하거나 무한 루프에 빠집니다.
### 개선 방식 : `low + (high - low) / 2`
- 장점 : 먼저 두 값의 차이`(high - low)`  를 구한 뒤 이를 2로 나누고 `low`에 더합니다. `high` 가 이미 정수 범위 내에 있다면 `high - 1ow`는 항상 안전하며, 최종 결과값도 `high` 를 넘지 않으므로 오버 플로 위험이 없습니다.
## 4. Lower Bound와 Upper Bound의 차이는? 각각 언제 사용하는가?

### Lower bound
![title](https://velog.velcdn.com/images/hyunsoo730/post/4cc6c33d-032f-4913-8225-763d83fc4d1d/image.png)   

- 정의 : 주어진 값 이상인 첫번째 원소의 위치를 반환
- 특정 값의 시작 위치를 찾는 알고리
- ex ) [1,3,3,5,7]에서 값3의 lower bound는 인덱스 1
- 동작 방식

초기에 left는 배열의 시작위치로 right는 배열의 길이로 세팅합니다
1. 배열의 중간값(mid)를 가져옵니다.
2. 중간 값과 검색 값을 비교합니다.
- 중간 값이 검색 값보다 작다면 left 값을 mid + 1로 합니다.
- 중간 값이 검색 값보다 크거나 같다면 right값을 mid로 합니다.
3. left >= right일 때까지 1번과 2번을 반복합니다.
4. 반복이 끝나면 right값이 lower bound가 됩니다.

▶ 특정값보다작은 첫번째 원소를 찾을 때 사용됩니다. 
### Upper bound
![title](https://velog.velcdn.com/images/hyunsoo730/post/8b27becb-47d7-4fc0-87e7-c98dc629c69f/image.png)   

- 정의 : 주어진 값보다 큰 첫번째원소의 위 치를반환
- 특정 K 값보다 처음으로 큰 값의 위치를 찾는 알고리즘입니다.
- ex) [1,3,3,5,7]에서 값3의 upper bound는 인덱스 3
- 동작 방식

초기에 left는 배열의 시작 위치, right는 배열의 길이로 세팅합니다.
1. 배열의 중간값(mid)를 가져옵니다.
2. 중간 값과 검색값을 비교합니다.
- 중간값이 검색값보다 작거나 같다면 left값을 mid + 1로 합니다.
- 중간값이 검색값도다 크다면 right 값을 mid로 합니다.
3. left >= right일 때까지 1번과 2번을 반복합니다.
4. 반복이 끝나면 right 값이 upper bound가 됩니다.

▶ 특정 값보다 작은 첫번째 원소를찾을 때 사용됩니다. 
## 5. 이진 탐색을 응용할 수 있는 문제 유형은? (parametric search)
### 5-1. 최적화 문제
- 정의: 어떤 제약 조건 내에서 목표로 하는 함수(목적 함수)의 값을 **최대화(Maximize)** 하거나 **최소화(Minimize)** 하는 변수의 값을 찾는 문제입니다.

### 5-2. 결정 문제
- 정의: 주어진 조건에 대해 **"예(Yes)"  혹은 "아니오(No)"** 로 답할 수 있는 질문으로 구성된 문제입니다.
- 최적화와의 관계: 최적화 문제의 "최댓값 x는 무엇인가?"를 **"x라는 값으로 조건을 만족할 수 있는가?"** 라는 질문으로 바꿉니다.
- 특징: 보통 `check(mid)`라는 별도의 함수를 만들어 해당 값이 가능한지 여부를 `Boolean`으로 반환하게 설계합니다.

### 5-3. 파라메트릭 서치
매개변수 탐색은 결정 문제의 답이 **단조성(Monotonicity)** 을 띨 때만 사용할 수 있습니다.
- 단조성 조건: 특정 지점을 기준으로 답이 `[True, True, True, False, False]`와 같이 일정한 방향으로 변해야 합니다. (이 경계선이 최적의 해가 됨)
- 작동 원리
1. 정답이 될 수 있는 **범위(`low`, `high`)** 를 설정합니다.
2. 중간값(mid)에 대해 **결정 문제(check(mid))** 를 실행합니다.
3. 결과가 `True` 라면 더 좋은 해를 찾기 위해 범위를 좁힙니다. (최댓값을 찾을 땐 오른쪽으로, 최솟값을 왼쪽으로)
4. 결과가 `False` 라면 정답이 불가능한 범위이므로 반대 방향으로 좁힙니다.
## 6. 회전된 정렬 배열에서 이진 탐색을 어떻게 적용하는가?
> 정렬된 배열이 특정 축을 기준으로 회전(Shift)된 경우에도 **O(log N)** 의 성능을 유지하며 값을 찾을 수 있습니다.
### 해결 전략 : 어느 쪽이 정렬돼 있는지 ?
- 배열이 회전되어도, `mid`를 기준으로 **왼쪽 절반 혹은 오른쪽 절반 중 하나는 반드시 정렬된 상태** 라는 성질을 이용합니다.
### 알고리즘 단계
1. `mid` 를 구합니다.
2. 왼쪽 부분 (`low` ~ `mid`)이 정렬되었는지 확인 : (`arr[low] <= arr[mid]`)
- `target` 이 그 범위 안에 있다면 ? ▷ `high = mid -1`
- 범위 밖이라면? ▷ 오른쪽 탐색 `low = mid + 1`
3. 위 조건이 아니면 오른쪽 부분(`mid` ~ `high`) 이 정렬된 것:
 - `target` 이 그 범위 안에 있다면 ? ▷ `low = mid +1`
- 범위 밖이라면? ▷ 오른쪽 탐색 `high = mid - 1`
