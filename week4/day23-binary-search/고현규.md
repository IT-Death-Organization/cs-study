# 1. 이진 탐색의 전제 조건은 무엇이며 왜 필요한가?

### 이진 탐색의 전제 조건
이진 탐색은 반드시 정렬되어있어야 합니다.

### 왜 필요한가?
이진 탐색이란 정렬된 데이터에서 특정 값을 찾기 위해 탐색 범위를 절반씩 좁혀나가는 효율적인 알고리즘입니다.

오름차순 정렬 데이터에서 중간값을 보고 내가 찾는 값보다 크면 왼쪽을, 작다면 오른쪽을 탐색 범위로 좁혀나갑니다.

그렇기 때문에 반드시 정렬되어 있어야 하고, 시간복잡도는 $O(\log N)$입니다.

만약 정렬이 되어있지 않다면 중간값을 보더라도 내가 찾는 값이 왼쪽에 있는지 오른쪽에 있는지 알 수 없습니다.

<br>

# 2. 이진 탐색이 O(log n)인 이유를 수학적으로 설명하면?

이진 탐색은 매 단계마다 탐색 범위를 절반($\frac{1}{2}$)씩 좁혀나갑니다.

$N$개의 데이터에 대해서 $k$번의 탐색을 수행한다고 했을 때

$N \times \frac{1}{2} \times \frac{1}{2} \times \cdots \times \frac{1}{2} = \frac{N}{2^k}$

이 때 $k$가 $\log_2 N$이 되어야 $\frac{N}{2^k}$이 1이 되므로 이진 탐색의 시간 복잡도는 $O(\log N)$입니다.

<br>

# 3. 이진 탐색 구현 시 mid 계산을 (low + high) / 2 대신 low + (high - low) / 2로 하는 이유는?

### (low + high) / 2 의 약점

프로그래밍 언어에서 `int`값이 가질 수 있는 최대 범위는 정해져 있습니다. 약 21억

만약 `low`와 `high`의 값이 매우 크다면 둘을 더했을 때 21억을 넘어갈 수 있고 overflow가 발생할 수 있습니다.

이를 방지하기 위해 `low + (high - low) / 2`로 계산합니다.

이 계산법은 수학적으로 동일하지만 overflow를 방지할 수 있습니다.

### `>>>` 연산자

```java
int mid = (low + high) >>> 1;
```

이런식으로 계산할 수도 있습니다.

`>>>`는 부호 없는 오른쪽 시프트(Unsigned Right Shift) 연산자입니다.

비트를 오른쪽으로 한 칸 이동시키면서, 가장 왼쪽(부호 비트)을 무조건 0으로 채웁니다.

low + high가 오버플로우되어 음수 비트 체계가 되었더라도, `>>> 1`을 하는 순간 부호 비트가 0이 되면서 정상적인 양수의 절반 값으로 돌아옵니다.

가장 빠르고 간결하지만, 자바 등 특정 언어에서만 가능합니다.

# 4. Lower Bound와 Upper Bound의 차이는? 각각 언제 사용하는가?

> Lower Bound와 Upper Bound는 중복이 허용된 정렬된 배열에서 타겟의 범위를 찾을 때 사용합니다.

### Lower Bound
Lower Bound는 찾고자 하는 값 이상($\ge$)의 데이터가 처음 나타나는 인덱스를 반환합니다.

예를 들어 배열에서 `arr[mid]`가 타겟보다 크거나 같으면 `high = mid`로 갱신하여 범위를 좁혀갑니다.

### Upper Bound
Upper Bound는 찾고자 하는 값보다 큰($>$) 데이터가 처음 나타나는 인덱스를 반환합니다.

예를 들어 배열에서 `arr[mid]`가 타겟보다 크면 `high = mid`로 갱신하여 범위를 좁혀갑니다.

### 언제 사용하는가?

정렬된 배열에서 특정 숫자의 갯수를 구하고 싶을 때 `upper bound - lower bound`를 사용합니다.

단순히 선형 탐색으로 숫자의 갯수를 센다면 시간 복잡도가 $O(N)$이지만, 이진 탐색으로 $O(\log N)$으로 줄일 수 있습니다.

실무적으로는 데이터베이스의 인덱스 스캔(Index Scan) 범위 결정이나, 특정 점수대(예: 80점~90점)의 학생 수를 계산할 때 가장 완벽한 방식입니다.

# 5. 이진 탐색을 응용할 수 있는 문제 유형은? (parametric search)

매개 변수 탐색(Parametric Search)은 최적화 문제를 결정 문제로 바꿔줍니다.

최적화 문제: 조건을 만족하는 최댓값(혹은 최솟값)은 얼마인가?

결정 문제: 이 값이 조건을 만족하는가? Yes or No

즉, 매개 변수 탐색은 결정 문제를 통해 Yes와 No의 경계선을 찾습니다.

### 예시
배가 7시간마다 고파지는 사람이 하루를 배부르게 지내기 위한 최소한의 식사횟수는?

이때 식사횟수를 24시간으로 나눴을 때 7에 가까운 값이 나와야 함을 알 수 있습니다.

0끼 ~ 10끼의 범위에서 이진 탐색을 수행합니다.

1단계: `(0 + 10) / 2` = 5끼 -> `24 / 5 = 4.8` < 7 -> 5끼는 많음.

2단계: `(0 + 4.9) / 2` = 2.4끼 -> `24 / 2.4 = 10` > 7 -> 2.4끼는 적음.

...

이 과정을 반복하면 시작값과 끝값의 대소관계가 역전되면서 원하는 값을 찾을 수 있습니다. 3.4끼

# 6. 회전된 정렬 배열에서 이진 탐색을 어떻게 적용하는가?

> 회전된 정렬 배열이란 `[0, 1, 2, 4, 5, 6, 7]`이 어떤 축을 기준으로 회전하여 `[4, 5, 6, 7, 0, 1, 2]`와 같이 된 상태를 말합니다.

잘 생각해보면 배열이 어떻게 회전되어 있어도 중간을 기준으로 어느 한 쪽은 정렬을 유지합니다. 쉽게 비유하자면 머리가 꼬리를 물고있는 지점은 단 한곳이기 때문입니다.

`arr[low] <= arr[mid]` → 왼쪽 절반이 정렬된 상태

`arr[low] > arr[mid]` → 오른쪽 절반이 정렬된 상태

이 개념을 유지한 채로 탐색을 합니다.

1. mid값을 통해 어느쪽이 정렬되어 있는지 먼저 확인합니다.
2. 정렬된 구간에 내가 찾는 값이 있는지 확인합니다.
    * 만약 있다면 정렬된 구간 안에서 이진 탐색을 수행합니다.
    * 없다면 반대편으로 넘어가서 다시 1번부터 수행합니다.
3. 이 과정을 반복하면 원하는 값을 찾을 수 있습니다.

이 방법을 통해 $O(\log N)$ 에 찾을 수 있습니다.
