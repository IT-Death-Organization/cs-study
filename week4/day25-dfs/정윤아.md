## Q1. DFS는 어떻게 동작하며 왜 깊이를 우선 탐색하는가?

[답변]
* DFS(Depth-First Search)
  - 현재 경로를 끝까지 추적한 후 다른 갈림길 확인
  - "깊이 우선": 같은 깊이의 다른 정점들을 먼저 확인 X, 현재 선택한 경로를 최대한 깊게 따라감

* 동작
  - 시작 정점 방문 처리
  - 방문하지 않은 인접 정점으로 이동
  - 해당 정점에서 같은 과정 반복
    - 방문 처리 + (아직 방문하지 않은)인접 정점 이동
  - 더 이상 이동할 곳이 없으면(방문하지 않은 인접 정점이 없으면) 되돌아감

## Q2. DFS를 스택으로 구현하는 이유는? 재귀로 구현하면 어떤 차이가 있는가?

[답변]
* 스택 구현 이유
  - 가장 나중에 방문 후보가 된 정점부터 다시 처리하는 구조
    => 후입선출 => 스택

* 재귀 구현
  - 내부적으로 함수 호출 스택 사용
  - 본질적으로 스택 기반 DFS와 동일한 원리

* 차이
  - 명시적 스택
    - 개발자가 직접 stack 자료구조 사용
    - 스택 상태 가시적으로 확인 가능
    - 깊은 탐색에서 호출 스택 초과 위험이 적음
    - 백트래킹 시 상태 함께 저장해야 함
      - 분기 처리를 직접 해줘야 함
      - 스택 내의 값만으로는 누가 누구의 인접 노드인지 알 수 없음
  - 재귀
    - 코드가 짧고 직관적
    - 함수 호출 자체가 스택 역할
    - 그래프가 매우 깊으면 재귀 깊이 제한 or 스택 오버플로 발생 가능
    - 백트래킹 구현이 쉬움
      - 자식 탐색이 모두 끝난 뒤 이후 코드 실행
      - 몇 번째 호출인지 알 수 있음
      - 자동으로 분기 처리가 됨
      - 함수 호출 자체가 인접 노드의 호출에 의한 것

## Q3. 재귀 DFS의 시간복잡도와 공간복잡도는? (그래프 기준)

[답변]
그래프를 인접 리스트로 표현한 경우

* 시간 복잡도
  - O(V + E)
  - V: 정점 수
  - E: 간선 수
  - 각 정점을 한 번씩 방문
  - 각 간선을 한 번씩 확인

* 공간 복잡도
  - O(V)
  - 방문 배열
  - 재귀 호출 스택

## Q4. DFS는 어떤 문제에 적합한가? 실제 활용 사례는?

[답변]
* 적합한 문제
  - 끝까지 파고들어야 구조를 파악할 수 있는 문제
  - ex
    - 그래프/트리 순회
    - 연결 요소 개수 세기
    - 경로 존재 여부 확인
    - 사이클 탐지
    - 위상 정렬
    - 미로 탐색
    - 조합, 순열, 부분집합 생성
    - 백트래킹 문제
  - 모든 경우의 수를 탐색해야 하는 경우
  - 특정 경로를 끝까지 확인해야 하는 경우
  - 탐색 도중 조건이 맞지 않으면 바로 되돌아가야 하는 경우

* 실제 활용 사례
  - 파일 시스템 디렉터리 탐색
  - 소셜 네트워크 연결 관계 분석
  - 웹 크롤링의 깊이 우선 접근
  - 퍼즐 문제 해 탐색
  - 컴파일러의 구문 트릐 순회
  - 게임에서 가능한 이동 경로 탐색

## Q5. 백트래킹과 DFS의 관계는? 백트래킹은 어떻게 구현하는가?

[답변]
* 백트래킹
  - DFS 기반 탐색 기법
  - 탐색 중인 경로가 정답이 될 수 없다고 판단 -> 더 깊이 들어가지 않고 즉시 되돌아 옴
  - DFS + 가지치기(pruning)
 
* 구현 방식
  - 재귀 DFS
  - 현재 상태에서 가능한 선택지 선택
    -> 조건 검사
    -> 유효하면 다음 단계 재귀 호출
    -> 탐색 후 선택 취소
    -> 이전 상태 복구

[보완]
* 일반 DFS visited: 전역적인 방문 완료 표시
* 백트래킹의 visited: 현재 경로에서 사용 중인지 표시

* 재귀 의사코드
```text
  dfs(depth):
    if depth == 목표깊이:
        결과 저장
        return

    for 후보 in 후보목록:
        if 방문 가능 또는 조건 만족:
            visited[후보] = true
            path에 후보 추가

            dfs(depth + 1)

            path에서 후보 제거
            visited[후보] = false
```

* 명시적 스택 의사코드
```text
  스택 <- [(depth=0, next_idx=0)]
  
  while 스택이 비어 있지 않으면:
      frame = 스택의 맨 위
  
      if frame.depth == 목표깊이:
          결과 저장
          스택에서 pop
          continue
  
      if frame.next_idx == 후보 개수:
          스택에서 pop
          if 스택이 비어 있지 않으면:
              부모선택 복구
          continue
  
      후보 = 후보목록[frame.next_idx]
      frame.next_idx += 1
  
      if 후보를 선택할 수 있으면:
          선택 반영
          스택에 (depth+1, next_idx=0) push
```

## Q6. DFS로 사이클을 탐지하는 방법은?

[답변]
* 무방향 그래프
  - 현재 정점에서 방문한 인접 정점을 다시 만남 -> 그 정점이 바로 직전 부모가 아니라면 사이클이 있다고 판단
  - 간선이 양방향: 자식에서 부모를 다시 보는 것은 당연 => 다시 만난 정점이 부모가 아닌 경우 사이클

* 방향 그래프
  - 탐색 중 현재 재귀 호출 경로에 들어 있는 정점 다시 만남 -> 사이클이 있다고 판단
  - visited: 방문 여부
  - recStack or inPath: 현재 DFS 경로 안에 있는지
  - DFS 탐색 중 inPath가 true인 정점을 다시 만나면 사이클
