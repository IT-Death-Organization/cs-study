## 🧠 학습 질문

## DFS는 어떻게 동작하며 왜 깊이를 우선 탐색하는가?

![image.png](attachment:1156b8bf-91d2-4c15-bf98-67bdb8bc8d0d:image.png)

- 시작 노드의 인접 정점 중 하나를 더 이상 탐색할 노드가 없을때까지 깊이 탐색하고,
- 더이상 탐색할 노드가 없으면 → 최근에 방문했던 노드로 퇴각 후 나머지 인접 노드도 같은 방식으로 탐색하는 알고리즘
- 스택 ( or  재귀 호출의 콜 스택)을 사용하여 그래프 탐색
- 한 번 방문한 노드는 다시 방문 X 끝까지 깊이 탐색 후 다시 돌아옴

![image.png](attachment:049634a0-3a3a-4fc9-8d9a-cfb184cf1a30:image.png)

- 왜 사용?
    - 현 경로상 노드를 기억함 → 적은 메모리 사용
    - 찾으려는 노드가 깊은 단계에 있으면 BFS보다 빠르게 찾음

## DFS를 스택으로 구현하는 이유는? 재귀로 구현하면 어떤 차이가 있는가?

- 스택 구현 이유
    - DFS는 가장 최근에 방문한 정점에서 다시 탐색을 시작해야 하므로 **LIFO(Last-In-First-Out)** 구조가 필요함
- 재귀로 하면
    - 구현이 더 편함. 재귀는 코드가 간결, 직관적
    - 재귀는 시스템의 스택 메모리를 사용하며, 그래프가 너무 깊으면 **Stack Overflow**가 발생할 수 있음 (스택은 힙메모리 사용 → 더 큰 그래프 처리 가능)
    - 재귀는 함수 호출 오버헤드가 있긴 한데, 대부분 미미한 수준

## 재귀 DFS의 시간복잡도와 공간복잡도는? (그래프 기준)

- 시간 복잡도
    - O(V+E) (정점 수 + 간선 수)
    - 모든 정점을 한 번씩 방문하고, 각 정점에서 연결된 모든 간선을 확인
- 공간 복잡도
    - O(V)
    - 모든 정점을 한 번씩 방문하고, 각 정점에서 연결된 모든 간선을 확인

## DFS는 어떤 문제에 적합한가? 실제 활용 사례는?

- 모든 경로를 탐색해야 하거나 특정 경로의 특징을 기록해야 할 때 유리
- 사례
    - **미로 찾기:** 특정 경로에 장애물이 있는지 끝까지 가봐야 하는 경우에 적합함
    - **위상 정렬(Topological Sort):** 작업 간의 우선순위를 결정할 때 사용함
    - **단절점/단절선 찾기:** 그래프의 연결 상태가 끊어지는 지점을 찾을 때 DFS 트리 개념을 활용함
    - **연결 요소(Connected Components) 찾기:** 맵에서 섬의 개수를 세는 등 붙어있는 정점 그룹을 찾을 때 쓰임

## 백트래킹과 DFS의 관계는? 백트래킹은 어떻게 구현하는가?

- DFS
    - 모든 노드를 방문하는 **탐색 알고리즘**
- 백트래킹
    - DFS를 기반으로 하되 불필요한 경로를 미리 차단(Pruning)하여 효율을 높이는 **전략**
    - 현재 노드가 유망하지 않다면 더 이상 깊이 들어가지 않고 부모 노드로 돌아감
    - 구현 방법
        - 상태 변경 (예: 체스판에 퀸 놓기)
        - 재귀 호출 (다음 단계 탐색)
        - 상태 복구

## DFS로 사이클을 탐지하는 방법은?

- 무방향 그래프
    - 방문했던 정점을 다시 방문했는데, 그 정점이 현재 노드의 부모노드가 아니라면 사이클이 존재하는 것임
- 유향 그래프
    - 현재 경로에서 아직 탐색이 끝나지 않은 노드(재귀 스택에 남아있는 노드)를 다시 만나면 사이클임
    - visited 배열 외에 is_in_recursion_stack 배열을 추가로 관리하여, 현재 깊이 탐색 중인 경로에 포함된 노드인지를 체크함
