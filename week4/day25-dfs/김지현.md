## 1. DFS는 어떻게 동작하며 왜 깊이를 우선 탐색하는가?

**DFS (Depth-First Search)** 는 그래프 또는 트리를 탐색할 때 **한 경로를 끝까지 탐색한 뒤 되돌아오는 방식**의 탐색 알고리즘이다.

동작 과정

1. 시작 노드를 방문한다.
2. 방문한 노드의 **인접 노드 중 아직 방문하지 않은 노드 하나를 선택**한다.
3. 그 노드로 이동하여 **같은 과정을 반복한다.**
4. 더 이상 방문할 노드가 없으면 **이전 노드로 돌아간다 (Backtracking)**.
5. 모든 노드를 방문할 때까지 반복한다.

예시 그래프

```
A
├─ B
│  ├─ D
│  └─ E
└─ C
```

DFS 탐색 순서

```
A → B → D → (back) → E → (back) → C
```

즉,

```
가능한 한 깊이 내려간 뒤
더 이상 갈 곳이 없으면 돌아온다
```

그래서 **Depth-First (깊이 우선)** 이라고 한다.

핵심 특징

-   한 경로를 **끝까지 탐색**
-   이후 **되돌아와 다른 경로 탐색**

# 2. DFS를 스택으로 구현하는 이유는? 재귀로 구현하면 어떤 차이가 있는가?

DFS의 핵심 구조는 **LIFO (Last-In First-Out)** 이다.

이 구조는 **스택(Stack)** 과 동일하다.

### 스택 기반 DFS

```
push(start)

while stack not empty
    node = stack.pop()
    if node not visited
        visit node
        push all neighbors
```

### 재귀 기반 DFS

```
dfs(node)
    visit node
    for neighbor in node
        if not visited
            dfs(neighbor)
```

재귀는 내부적으로 **Call Stack** 을 사용한다.

즉

```
재귀 DFS = 암묵적 스택
스택 DFS = 명시적 스택
```

차이

| 구분                | 재귀 DFS | 스택 DFS  |
| ------------------- | -------- | --------- |
| 구현 난이도         | 간단     | 조금 복잡 |
| 가독성              | 높음     | 낮음      |
| Stack Overflow 위험 | 있음     | 없음      |
| 메모리 제어         | 어려움   | 가능      |

실무에서는

```
그래프 크기가 매우 크면
재귀 대신 stack DFS 사용
```

# 3. 재귀 DFS의 시간복잡도와 공간복잡도는? (그래프 기준)

그래프

```
V = vertex 수
E = edge 수
```

### 시간복잡도

DFS는

```
각 정점 1번 방문
각 간선 최대 1번 탐색
```

따라서

```
O(V + E)
```

이유

```
모든 vertex 방문 → O(V)
모든 edge 검사 → O(E)
```

---

### 공간복잡도

공간 사용

1. visited 배열
2. recursion stack
3. adjacency list

DFS 자체 스택

```
최대 깊이 = V
```

따라서

```
공간복잡도

O(V)
```

(그래프 저장 제외)

# 4. DFS는 어떤 문제에 적합한가? 실제 활용 사례는?

DFS는 다음 문제에 매우 적합하다.

### 1. 연결 요소 탐색 (Connected Components)

```
그래프가 몇 개의 컴포넌트로 나뉘는가
```

예

-   네트워크 연결
-   섬 개수 문제

---

### 2. 경로 탐색

```
특정 노드에서 다른 노드까지 경로 존재 여부
```

예

-   미로 탐색
-   게임 맵 탐색

---

### 3. 위상 정렬 (Topological Sort)

DAG에서

```
작업 순서 결정
```

예

-   빌드 시스템
-   컴파일 의존성

---

### 4. 사이클 탐지

그래프에

```
cycle 존재 여부 확인
```

예

-   Deadlock detection
-   dependency graph

---

### 5. 백트래킹 문제

대표 예시

-   N-Queen
-   Sudoku
-   Permutation / Combination 생성

---

# 5. 백트래킹과 DFS의 관계는? 백트래킹은 어떻게 구현하는가?

**백트래킹 = DFS의 한 형태**

차이

```
DFS = 모든 경로 탐색

백트래킹 = 조건을 만족하지 않으면 탐색 중단
```

즉

```
DFS + Pruning
```

구조

```
dfs(state)

if invalid
    return

if solution
    record

for next_state
    dfs(next_state)
```

예: N-Queen

```
row마다 queen 배치

dfs(row)
    if row == N
        solution

    for col in 0..N
        if safe
            place queen
            dfs(row+1)
            remove queen
```

여기서

```
remove queen
```

이 과정이 **Backtracking** 이다.

# 6. DFS로 사이클을 탐지하는 방법은?

그래프 유형에 따라 다르다.

---

## 1️⃣ Undirected Graph

사이클 조건

```
이미 방문한 노드인데
부모(parent)가 아니라면 cycle
```

코드 구조

```
dfs(node, parent)

visited[node] = true

for neighbor in adj[node]
    if not visited
        dfs(neighbor, node)

    else if neighbor != parent
        cycle 발견
```

---

## 2️⃣ Directed Graph

Directed graph에서는

```
현재 DFS 경로에 있는 노드를 다시 방문하면 cycle
```

그래서 **3가지 상태**를 사용한다.

```
0 = not visited
1 = visiting
2 = visited
```

코드

```
dfs(node)

state[node] = visiting

for neighbor
    if state[neighbor] == visiting
        cycle

    if state[neighbor] == not visited
        dfs(neighbor)

state[node] = visited
```

이 방식은

```
Recursion Stack detection
```

이라고 한다.

# 정리

| 항목        | 핵심                                                |
| ----------- | --------------------------------------------------- |
| DFS 특징    | 한 경로를 끝까지 탐색                               |
| 구현        | Stack 또는 Recursion                                |
| 시간복잡도  | O(V + E)                                            |
| 공간복잡도  | O(V)                                                |
| 적합 문제   | 경로 탐색, 연결 요소, 위상 정렬, 사이클 탐지        |
| 백트래킹    | DFS + pruning                                       |
| 사이클 탐지 | undirected: parent 확인 / directed: recursion stack |
