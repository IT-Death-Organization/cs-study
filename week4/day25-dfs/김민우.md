# DFS (Depth First Search)

## 1. DFS는 어떻게 동작하며 왜 깊이를 우선 탐색하는가?

**DFS (Depth First Search)** 는 그래프 또는 트리에서 **한 경로를 가능한 한 끝까지 탐색한 후, 더 이상 진행할 수 없으면 이전 노드로 돌아가 다른 경로를 탐색하는 방식의 알고리즘**이다.

### 탐색 방식

1. 시작 노드를 방문한다.
2. 인접한 노드 중 **아직 방문하지 않은 노드로 이동**한다.
3. 그 노드에서도 동일하게 **계속 깊게 들어간다.**
4. 더 이상 방문할 노드가 없으면 **이전 노드로 되돌아간다 (Backtracking)**.
5. 다른 인접 노드가 있으면 다시 탐색을 진행한다.

### 왜 깊이를 우선 탐색하는가?

DFS는 다음 원리에 따라 동작한다.

* **현재 경로를 끝까지 탐색하는 것이 효율적인 문제들이 존재**
* 스택 구조 때문에 **최근 방문 노드부터 확장**

예시 그래프

```
A
├── B
│   ├── D
│   └── E
└── C
```

DFS 탐색 순서

```
A → B → D → (back) → E → (back) → C
```

즉 **가능한 한 깊이 들어갔다가 돌아오는 방식**이다.

---

# 2. DFS를 스택으로 구현하는 이유

DFS의 핵심 원리는

> **"가장 최근에 방문한 노드부터 다시 탐색한다"**

이 구조는 **LIFO (Last In First Out)** 구조와 동일하다.

따라서 **스택(Stack)** 을 사용하면 DFS를 자연스럽게 구현할 수 있다.

### 스택 기반 DFS

```java
Stack<Integer> stack = new Stack<>();
stack.push(start);

while(!stack.isEmpty()){
    int node = stack.pop();

    if(!visited[node]){
        visited[node] = true;

        for(int next : graph[node]){
            stack.push(next);
        }
    }
}
```

# 3. 재귀 DFS와 스택 DFS 차이

## (1) 재귀 DFS

재귀는 **함수 호출 스택(call stack)** 을 사용한다.

```java
void dfs(int node){
    visited[node] = true;

    for(int next : graph[node]){
        if(!visited[next]){
            dfs(next);
        }
    }
}
```

### 특징

* 코드가 **매우 간결**
* 백트래킹 구현이 쉬움
* 하지만 **재귀 깊이가 깊으면 StackOverflow 가능**

---

## (2) 스택 DFS

직접 스택을 관리한다.

### 특징

* 재귀 깊이 제한 없음
* 메모리 관리가 명확
* 코드가 조금 더 복잡

---

## 차이 요약

| 구분    | 재귀 DFS        | 스택 DFS       |
| ----- | ------------- | ------------ |
| 구현    | 간단            | 조금 복잡        |
| 메모리   | Call Stack 사용 | 명시적 Stack 사용 |
| 재귀 깊이 | 제한 있음         | 제한 없음        |
| 백트래킹  | 자연스럽게 가능      | 직접 구현 필요     |

---

# 4. DFS 시간복잡도 / 공간복잡도 (그래프 기준)

그래프가

* **V : 정점 수**
* **E : 간선 수**

일 때

## 시간복잡도

```
O(V + E)
```

이유

* 모든 정점 **한 번 방문**
* 모든 간선 **한 번 탐색**

---

## 공간복잡도

### 방문 배열

```
O(V)
```

### 재귀 스택

최악의 경우

```
O(V)
```

따라서

```
총 공간복잡도 = O(V)
```

# 5. DFS가 적합한 문제 유형

DFS는 **경로 탐색 / 모든 경우 탐색** 문제에 강하다.

## 대표 문제 유형

### 1️⃣ 그래프 탐색

* 연결 요소 찾기
* 그래프 순회

예

* Connected Components

---

### 2️⃣ 모든 경로 탐색

* 가능한 경우의 수 찾기

예

* 미로 탐색
* 경로 찾기

---

### 3️⃣ 조합 / 순열 문제

예

* N-Queens
* 부분집합
* 순열 생성

---

### 4️⃣ 그래프 구조 분석

예

* 사이클 탐지
* 위상 정렬
* SCC (Strongly Connected Components)

# 6. 백트래킹과 DFS 관계

백트래킹은 **DFS의 확장 개념**이다.

### DFS

```
모든 경로를 끝까지 탐색
```

### 백트래킹

```
조건을 만족하지 않으면 탐색을 중단하고 되돌아감
```

즉

```
DFS + 가지치기(pruning)
```

---

## 백트래킹 구조

```java
void dfs(int depth){

    if(종료조건){
        결과 저장
        return;
    }

    for(선택지){
        if(가능한 선택){
            선택

            dfs(depth + 1)

            선택 취소
        }
    }
}
```

### 핵심

```
선택 → 탐색 → 취소
```

---

## 예시 (순열)

```java
void dfs(int depth){

    if(depth == n){
        print();
        return;
    }

    for(int i=0;i<n;i++){
        if(!visited[i]){
            visited[i] = true;
            arr[depth] = i;

            dfs(depth+1);

            visited[i] = false;
        }
    }
}
```

# 7. DFS로 사이클 탐지 방법

그래프에서 사이클을 찾는 방법은 그래프 종류에 따라 다르다.

---

# 7.1 무방향 그래프 사이클 탐지

부모 노드를 이용한다.

핵심 아이디어

```
이미 방문한 노드인데
부모가 아니면 사이클
```

### 코드

```java
boolean dfs(int node, int parent){

    visited[node] = true;

    for(int next : graph[node]){

        if(!visited[next]){
            if(dfs(next, node)) return true;
        }

        else if(next != parent){
            return true;
        }
    }

    return false;
}
```

---

# 7.2 방향 그래프 사이클 탐지

방문 상태를 3가지로 관리한다.

```
0 = 방문 안함
1 = 방문 중
2 = 방문 완료
```

### 핵심

```
방문 중 노드를 다시 만나면 사이클
```

### 코드

```java
boolean dfs(int node){

    visited[node] = 1;

    for(int next : graph[node]){

        if(visited[next] == 0){
            if(dfs(next)) return true;
        }

        else if(visited[next] == 1){
            return true;
        }
    }

    visited[node] = 2;
    return false;
}
```
