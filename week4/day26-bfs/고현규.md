# 1. BFS는 어떻게 동작하며 왜 너비를 우선 탐색하는가?

> BFS(Breadth-First Search)는 그래프나 트리를 시작 노드에서 가까운 노드부터 차례대로 탐색하는 알고리즘입니다.

### BFS 동작 방식
1. **시작 노드 삽입**
   - 시작 노드를 큐(Queue)에 넣고 방문 처리합니다.
2. **같은 레벨(너비) 먼저 탐색**
   - 큐에서 노드를 하나 꺼냅니다.
   - 그 노드의 인접 노드 중 아직 방문하지 않은 노드를 모두 큐에 넣고 방문 처리합니다.
   - 큐가 빌 때까지 이 과정을 반복합니다.
3. **FIFO를 이용해서 가까운 노드부터 탐색**
   - 큐는 선입선출(FIFO)이므로, 먼저 들어간 노드, 더 가까운 노드가 먼저 나옵니다.
   - 그래서 항상 현재 깊이(레벨)를 다 탐색한 뒤에 다음 깊이로 넘어가게 됩니다.

### 탐색 순서

![BFS 트리 탐색 순서 이미지](https://kayuse88.github.io/assets/img/posts/dfs-bfs/Breadth-first-tree.svg)

### 너비를 우선 탐색하는 이유

큐의 FIFO 특성으로 인해 시작점에서 가까운 노드(같은 레벨)를 먼저 모두 처리하고, 그 다음 레벨로 넘어가기 때문에 너비가 우선됩니다.

가중치 없는 그래프에서의 최단 경로(최단 거리)를 구할 때 BFS가 적합합니다.

# 2. BFS를 큐로 구현하는 이유는? 스택으로 구현하면 어떻게 되는가?

BFS는 "현재 노드에서 가까운 노드"부터 하나씩 넓게 퍼져나가는 방식이기 때문에, 먼저 들어간 노드가 먼저 나오는 큐(FIFO)와 맞습니다. 스택으로 구현하면 깊이 우선 탐색처럼 동작해 BFS가 아닙니다.

### 큐를 쓰는 이유

큐는 선입선출(FIFO) 자료구조라서, 먼저 큐에 들어간(즉, 시작점과 더 가까운) 노드가 먼저 탐색됩니다. 같은 "깊이/거리"에 있는 모든 노드를 탐색한 뒤에 다음 깊이로 넘어갈 수 있습니다.

예를 들어 시작 노드 A에서 BFS를 진행한다면 큐에 A를 넣고, A를 꺼낸 뒤 인접한 B, C, D를 큐에 넣습니다. 다음에는 B, C, D 순서대로 꺼내며 각각의 인접 노드를 추가하게 됩니다. 이 덕분에 "처음 도착"한 시점이 곧 "최단 거리"로 도착한 순간이 되어 최단 경로 탐색에 유리합니다.

### 스택으로 구현하면?

스택은 LIFO 구조입니다. 새로 추가된(최근 인접) 노드부터 계속 탐색하게 되므로 사실상 DFS(깊이 우선 탐색)처럼 동작합니다. BFS의 핵심인 "거리 순서" 탐색이 깨지고, 최단 경로를 보장하지 못합니다. 정리하면, BFS의 "너비 우선, 거리 순서" 성질을 살리려면 큐가 필요합니다.

### 코드 예시 (파이썬)

```python
from collections import deque

def bfs(graph, start):
    visited = set()
    queue = deque([start])
    visited.add(start)

    while queue: # 큐가 빌 때까지
        node = queue.popleft()
        # 노드 처리
        for neighbor in graph[node]:
            if neighbor not in visited:
                queue.append(neighbor) # 현재 탐색중인 노드의 자식들을 큐의 "뒤"에 밀어넣기
                visited.add(neighbor) # 큐에 추가되어 탐색을 기다리는 노드가 혹시 뒤에 또 추가되는걸 방지하기 위해 방문처리
```

`queue.popleft()`로 가장 먼저 들어간(=가까운) 노드부터 처리합니다. 만약 `pop()`만 쓴다면 최근에 추가된 노드부터 탐색하게 되어 DFS처럼 동작합니다.

# 3. BFS의 시간복잡도와 공간복잡도는? (그래프 기준)

그래프의 노드의 개수를 V, 간선의 개수를 E라고 할 때 복잡도를 알아보겠습니다.

### 시간복잡도

BFS도 DFS와 마찬가지로 모든 노드를 한 번씩 방문하고, 모든 간선을 한 번씩 검사하는 것이 목표입니다.

각 노드는 큐에 한 번 들어가고 한 번 나옵니다. 방문 여부(`visited`) 체크로 한 번만 큐에 들어가므로 노드 처리에 $O(V)$입니다.

각 노드에서 연결된 인접 노드를 확인해 큐에 넣습니다. 인접 리스트 기준으로 모든 간선을 정확히 한 번씩 보게 되므로 $O(E)$입니다.

전체 시간복잡도는 $O(V+E)$입니다.

### 공간복잡도

BFS는 큐에 "다음에 탐색할 노드들"을 쌓아둡니다. 최악의 경우 시작 노드와 같은 레벨에 있는 노드가 매우 많을 때(예: 완전 이진 트리의 마지막 레벨처럼) 그 수만큼 큐에 들어갑니다.

그래프가 인접 리스트로 주어졌을 때, 큐에 동시에 들어갈 수 있는 노드 수는 최악에 $O(V)$까지 갈 수 있으므로 공간복잡도는 $O(V)$입니다. visited 배열 등 보조 자료도 $O(V)$이므로 전체 공간복잡도는 $O(V)$로 봅니다.

# 4. 최단 경로 문제에서 BFS를 사용하는 이유는?

> BFS는 가중치가 없는 그래프에서 "간선 개수 기준 최단 경로(최단 거리)"를 보장합니다.

BFS는 시작점에서 가까운 노드(같은 거리/레벨)를 먼저 모두 탐색한 뒤, 그 다음 거리의 노드를 탐색합니다.

따라서 어떤 노드에 처음 도달하는 순간이 곧 "그 노드까지의 간선 개수가 가장 적은 경로로 도달한 순간"이 됩니다.  
이후 같은 노드로 다른 경로로 다시 도착해도, 이미 방문 처리되어 있으므로 더 먼 경로는 무시됩니다.

### 활용 예

1. **미로에서 최단 거리**
    - 칸을 노드, 상하좌우를 간선으로 두면 BFS로 출발→도착 최단 칸 수를 구할 수 있습니다.
2. **레벨/거리 기록**
    - BFS 탐색 시 각 노드를 "몇 번째 레벨에서 방문했는지" 저장하면, 그 값이 곧 시작점부터의 최단 거리(간선 개수)가 됩니다.

참고로 간선에 가중치(비용)가 있는 그래프에서는 BFS로는 최단 경로를 보장할 수 없고, 다익스트라 등 다른 알고리즘이 필요합니다.

# 5. DFS와 BFS를 언제 선택해야 하는가? 각각의 장단점은?

> 문제가 "거리/레벨 순서"를 요구하면 BFS, "경로 하나를 끝까지" 또는 "모든 경우의 수"를 요구하면 DFS를 고려합니다.

### BFS를 선택하는 경우

1. 최단 경로(간선 개수 기준)를 구할 때 (가중치 없는 그래프)
2. 레벨 순서대로 노드를 처리해야 할 때 (예: 트리 레벨 순회)
3. 시작점에서 가까운 것부터 퍼져 나가는 시뮬레이션 (예: 바이러스 확산, 다리 놓기)

### DFS를 선택하는 경우 (day 25 참고)

1. 경로 존재 여부만 확인하면 될 때 (최단이 아니어도 됨)
2. 사이클 탐지, 위상 정렬 등 그래프 구조 분석
3. 모든 경우의 수/조합 탐색 (백트래킹, N-Queens, 스도쿠 등)
4. 메모리가 매우 제한적일 때 (DFS는 경로만 스택에 쌓아서 공간이 적을 수 있음)

### 장단점 요약

| 구분 | BFS | DFS |
| :--- | :--- | :--- |
| **장점** | 최단 경로(비가중) 보장, 레벨 순서 보장 | 구현 간단(재귀), 메모리 적을 수 있음, 백트래킹에 적합 |
| **단점** | 큐로 인해 공간 사용이 많을 수 있음 | 최단 경로 보장 안 됨, 깊이 제한 시 스택 오버플로우 가능 |

# 6. 가중치가 있는 그래프에서는 BFS로 최단 경로를 못 구하는 이유는?

> BFS는 간선을 몇 개 거쳤는지만 구분할 뿐, 간선의 비용(가중치) 는 보지 않습니다.

BFS가 보는 것은 거리, 즉 간선의 갯수 뿐입니다.

BFS는 큐에 넣는 순서가 "시작점에서 간선을 타고 몇 번만에 도달했는지"에만 의존합니다.  
즉, 모든 간선을 길이 $1$로 취급합니다.

그래서 간선 2개를 타고 도착한 경로가 간선 1개만 타고 도착한 경로보다 항상 나중에 처리됩니다.  

### 가중치가 있으면 왜 깨지는가?

가중치가 있으면 간선 개수가 적어도 총 비용이 더 클 수 있습니다.

예를 들어 A → B(비용 10), A → C(비용 1), C → B(비용 1)인 그래프를 생각해 봅시다. BFS는 A → B를 "1번에 도착"으로 보고 먼저 처리하고, A → C → B는 "2번에 도착"으로 나중에 처리합니다. 하지만 실제 최단 비용은 A → C → B = 2입니다. BFS는 몇 번째에 도착했는지만 보기 때문에 비용이 작은 경로를 우선할 수 없습니다.

### 참고

간선에 가중치가 있을 때의 최단 경로는 다익스트라(Dijkstra), 벨만-포드 등 비용을 고려하는 알고리즘을 사용해야 합니다. 이들은 "거리 = 간선 가중치의 합"을 최소화합니다.
