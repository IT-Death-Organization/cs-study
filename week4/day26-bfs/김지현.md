# 1. BFS는 어떻게 동작하며 왜 너비를 우선 탐색하는가?

**BFS (Breadth-First Search)** 는
그래프에서 **시작 노드로부터 가까운 노드부터 차례대로 탐색하는 알고리즘**입니다.

탐색 순서의 핵심은 **level(거리)** 입니다.

```
Level 0 : 시작 노드
Level 1 : 시작 노드와 직접 연결된 노드
Level 2 : Level1 노드들과 연결된 노드
Level 3 : ...
```

예시 그래프

```
    A
   / \
  B   C
 / \   \
D   E   F
```

BFS 탐색 순서

```
A → B → C → D → E → F
```

즉

```
거리 0 → 거리 1 → 거리 2 → ...
```

순서로 탐색합니다.

### 왜 "너비 우선"인가?

트리나 그래프를 **층(level)** 단위로 확장하기 때문입니다.

```
DFS : 한 방향으로 끝까지 내려감
BFS : 같은 깊이(level)의 노드를 모두 탐색
```

---

# 2. BFS를 큐로 구현하는 이유는? 스택으로 구현하면?

BFS는 **먼저 발견한 노드를 먼저 탐색해야 합니다.**

즉 **FIFO (First In First Out)** 구조가 필요합니다.

그래서 **Queue**를 사용합니다.

## BFS 동작 과정

```
1 시작 노드 방문
2 큐에 삽입
3 큐에서 노드 꺼냄
4 인접 노드 큐에 삽입
5 반복
```

의사 코드

```
BFS(start):

queue.push(start)
visited[start] = true

while queue not empty:

    node = queue.pop()

    for next in node.adjacent:
        if not visited[next]:
            visited[next] = true
            queue.push(next)
```

---

### 스택으로 구현하면?

스택은 **LIFO (Last In First Out)** 입니다.

그래서

```
최근 노드 → 먼저 탐색
```

이 됩니다.

결과

```
DFS가 됨
```

즉

```
Queue → BFS
Stack → DFS
```

---

# 3. BFS의 시간복잡도와 공간복잡도 (그래프)

그래프를

```
V = 정점 수
E = 간선 수
```

라고 하면

### 시간복잡도

```
O(V + E)
```

이유

1. 모든 정점은 **한 번씩 방문**
2. 모든 간선은 **한 번씩 확인**

그래프 탐색 알고리즘의 기본 복잡도입니다.

---

### 공간복잡도

```
O(V)
```

이유

저장해야 하는 것

```
visited 배열
queue
```

최악의 경우

```
queue에 V개의 노드 저장
```

---

# 4. 최단 경로 문제에서 BFS를 사용하는 이유

**BFS는 "간선 수 기준 최단 경로"를 보장합니다.**

이유는 **level 탐색 구조** 때문입니다.

예시

```
A → B → D
A → C → E → D
```

BFS 탐색

```
A
B C
D E
```

D는

```
A → B → D (2 edges)
```

로 먼저 발견됩니다.

핵심 원리

```
BFS는 거리 k인 노드를
거리 k+1보다 먼저 방문한다
```

그래서

```
unweighted graph
```

에서는

```
최단 경로 알고리즘
```

이 됩니다.

---

# 5. DFS와 BFS는 언제 선택하는가?

| 항목      | BFS               | DFS               |
| --------- | ----------------- | ----------------- |
| 탐색 방식 | level 탐색        | 깊이 탐색         |
| 자료구조  | queue             | stack / recursion |
| 최단 경로 | 가능 (unweighted) | 불가능            |
| 메모리    | 큼                | 작음              |
| 구현      | 조금 복잡         | 간단              |

---

### BFS 사용하는 경우

```
최단 거리
최소 이동 횟수
레벨 탐색
```

대표 문제

```
미로 탐색
최단 경로
네트워크 거리
최소 단계 문제
```

---

### DFS 사용하는 경우

```
경로 탐색
모든 경우 탐색
트리 구조 분석
```

대표 문제

```
백트래킹
사이클 탐지
위상 정렬
연결 요소 찾기
```

---

# 6. 가중치 그래프에서 BFS로 최단 경로를 못 구하는 이유

BFS는 **간선 수 기준으로만 탐색**합니다.

하지만 가중치 그래프에서는

```
비용(cost)
```

이 중요합니다.

예시

```
A → B (cost 100)
A → C (cost 1)
C → B (cost 1)
```

BFS 탐색

```
A → B
```

간선 수

```
1
```

하지만 실제 최단 경로

```
A → C → B
cost = 2
```

그래서 BFS는 틀린 결과를 냅니다.

---

### 그래서 사용하는 알고리즘

| 그래프      | 알고리즘     |
| ----------- | ------------ |
| 가중치 없음 | BFS          |
| 가중치 있음 | Dijkstra     |
| 음수 가중치 | Bellman-Ford |

---

# 핵심 요약

BFS의 본질

```
Queue 기반 Level 탐색
```

시간복잡도

```
O(V + E)
```

특징

```
간선 수 기준 최단 경로 보장
```

그래서

```
Unweighted shortest path 문제에서 사용
```

하지만

```
Weighted graph에서는 사용할 수 없음
```
