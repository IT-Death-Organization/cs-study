## 1. BFS는 어떻게 동작하며 왜 너비를 우선 탐색하는가?

**인접 노드 우선 방문**
- 시작 노드에서 출발하여 인접한 모든 노드를 우선적으로 방문한 뒤, 방문했던 노드들을 기준으로 다시 그 인접 노드들을 차례대로 탐색하는 방식이다.
- **너비 우선의 이유:** 현재 위치에서 가장 가까운 노드들을 먼저 모두 확인하여 탐색의 범위를 넓혀가기 위함이다. 마치 호수에 돌을 던졌을 때 파동이 사방으로 일정하게 퍼져나가는 것과 같은 원리로 동작한다.



---

## 2. BFS를 큐로 구현하는 이유는? 스택으로 구현하면 어떻게 되는가?

**FIFO 구조의 필요성**
- **큐(Queue) 사용 이유:** 먼저 발견한 노드를 먼저 방문해야 하는 **선입선출(FIFO)** 구조가 너비 우선 탐색의 논리와 완벽히 일치하기 때문이다. 예약된 순서대로 노드를 꺼내어 방문함으로써 레벨 단위의 탐색을 보장한다.
- **스택(Stack)으로 구현할 경우:** 가장 나중에 발견한 노드를 먼저 방문하게 되므로, 이는 BFS가 아닌 **DFS(깊이 우선 탐색)**의 동작 방식이 된다. 즉, 탐색의 방향이 옆으로 퍼지지 않고 아래로 깊게 파고들게 된다.

---

## 3. BFS의 시간복잡도와 공간복잡도는? (그래프 기준)

- **시간 복잡도:** 모든 정점($V$)을 큐에 한 번씩 넣고 빼며, 각 정점마다 연결된 모든 간선($E$)을 확인한다. 따라서 인접 리스트 기준 **$O(V + E)$**가 소요된다. (인접 행렬 기준은 $O(V^2)$)
- **공간 복잡도:** 방문 여부를 기록하기 위한 배열과 더불어, 최악의 경우 큐에 한꺼번에 많은 정점이 담길 수 있으므로 **$O(V)$**가 된다.

---

## 4. 최단 경로 문제에서 BFS를 사용하는 이유는?

**최단 거리의 보장**
- BFS는 시작 노드로부터 거리가 1인 노드들, 거리가 2인 노드들 순으로 레벨을 높여가며 탐색한다.
- 따라서 목표 노드를 **처음 발견한 순간**이 곧 시작점으로부터 해당 노드까지 도달하는 **가장 짧은 경로**임이 수학적으로 보장된다. 깊이 우선으로 탐색하는 DFS와 달리, 멀리 돌아가는 경로를 먼저 확인하지 않기 때문에 가중치가 없는 그래프에서 최단 거리를 찾는 데 최적이다.

## 5. DFS와 BFS를 언제 선택해야 하는가? 각각의 장단점은?

| 구분 | DFS (깊이 우선 탐색) | BFS (너비 우선 탐색) |
| :--- | :--- | :--- |
| **장점** | 공간을 적게 차지하며, 목표 노드가 깊을 때 유리함 | 최단 경로를 보장하며, 목표 노드가 시작점 근처일 때 유리함 |
| **단점** | 최단 경로를 보장하지 못하며, 무한 루프 위험이 있음 | 노드가 많을 경우 큐에 많은 데이터가 쌓여 메모리 부담이 큼 |
| **선택 기준** | 모든 경로를 탐색하거나 백트래킹이 필요한 경우 | 가중치 없는 그래프에서 최단 경로를 찾아야 하는 경우 |



---

## 6. 가중치가 있는 그래프에서는 BFS로 최단 경로를 못 구하는 이유는?

**단순 방문의 한계**
- BFS는 시작점으로부터의 '간선의 개수'를 기준으로 탐색한다. 하지만 가중치가 있는 그래프에서는 간선이 1개인 경로보다 간선이 2~3개인 경로의 **가중치 합**이 더 작을 수 있다.
- BFS는 노드를 한 번 방문하면 최단 거리라고 확정 짓고 다시 방문하지 않기 때문에, 나중에 발견된 더 짧은(가중치 합이 작은) 경로를 갱신하지 못한다. 이를 해결하기 위해 가중치 그래프에서는 **다익스트라(Dijkstra)**와 같은 알고리즘을 사용한다.
