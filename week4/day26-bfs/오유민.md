### 📌 [Q1] BFS는 어떻게 동작하며 왜 너비를 우선 탐색하는가?

BFS는 **시작 노드에서 출발해 거리가 같은 노드들을 먼저 모두 방문한 뒤, 그 다음 거리의 노드들을 방문하는 방식**이다.

**BFS의 핵심 메커니즘: 선입선출(FIFO) 구조의 큐(Queue)**
- BFS의 순서를 보장하기 위해 큐의 메모리 구조를 사용한다.
- (1) 시작 노드 방문: 시작 노드를 방문 처리하고 큐에 넣는다.
- (2) 반복 탐색: 큐가 빌 때까지 다음 과정을 반복한다.
  - 큐에서 노드 하나 꺼냄 (Dequeue)
  - 꺼낸 노드의 인접 노드 중 아직 방문하지 않은 노드들을 모두 확인함
  - 그 노드들을 방문 처리하고 큐에 차례대로 넣음 (Enqueue)

**동작 과정**<br>
- 시작 노드 방문: 탐색을 시작할 노드를 방문하고 '방문 처리'를 합니다.
- 인접 노드 확인: 현재 노드와 연결된 방문하지 않은 모든 인접 노드를 큐에 삽입합니다.
- 종료: 모든 노드를 방문할 때까지(큐가 빌 때까지) 반복합니다.

**왜 너비를 우선 탐색하는가?**
- **최단 경로 보장**
  - BFS는 시작 노드로부터 **가까운 노드부터 차례대로 방문**하기 때문에, **가중치가 없는 그래프에서 최단 경로를 보장**한다.
  - BFS는 목표 노드를 발견하는 순간이 곧 최단 경로임을 보장함
- **계층 구조 파악**
  - 노드 간의 "거리"나 "단계"가 중요한 문제에서 유리함 (ex. SNS 1촌, 2촌 찾기 등)
- **무한한 깊이 방지**
  - 그래프가 매우 깊을 때 DFS는 스택 오버플로우가 발생하거나 정답을 찾는 데 너무 오래 걸릴 수 있지만, BFS는 정답이 시작점 근처라면 훨씬 빠르게 찾음

### 📌 [Q2] BFS를 큐로 구현하는 이유는? 스택으로 구현하면 어떻게 되는가?

**큐(Queue)로 구현하는 이유**
- **선입선출(FIFO)의 정합성**: BFS는 먼저 발견된 노드를 먼저 방문해야 하는 '너비 순서'를 지켜야 한다. 큐는 먼저 들어온 데이터를 먼저 처리하므로 이 논리에 완벽히 부합한다.
- **공평한 기회 부여**: 한 노드의 이웃들을 모두 큐에 넣으면, 그 이웃들이 다음 방문 후보가 되어 공평하게 한 레벨씩 확장해 나갈 수 있다.

**스택(Stack)으로 구현하면?**
- **DFS(깊이 우선 탐색)가 됨**: 스택의 LIFO(후입선출) 특성 때문에 가장 최근에 발견된 노드를 먼저 방문하게 되어, 옆으로 퍼지는 것이 아니라 한 방향으로 깊게 파고드는 DFS 방식으로 동작하게 된다.
- **최단 거리 보장 불가**: 더 이상 거리순으로 탐색하지 않게 되므로, 처음 발견한 목표 노드가 최단 거리에 있다는 보장이 사라진다.

### 📌 [Q3] BFS의 시간복잡도와 공간복잡도는? (그래프 기준)

**시간 복잡도: O(V + E)** (인접 리스트 기준)
- 모든 노드(V)를 한 번씩 방문하고, 각 노드에 연결된 모든 간선(E)을 한 번씩 검사하므로 O(V + E)가 된다.
- 인접 행렬로 구현할 경우, 모든 노드마다 다른 모든 노드와의 연결을 확인해야 하므로 **O(V²)**이 된다.

**공간 복잡도: O(V)**
- **방문 배열**: 모든 노드의 방문 여부를 관리하기 위해 O(V)의 공간이 필요하다.
- **큐(Queue)**: 최악의 경우(예: 시작 노드에 수많은 노드가 직접 연결됨) 큐에 모든 노드 정보가 들어갈 수 있어 O(V)의 공간을 사용한다.

### 📌 [Q4] 최단 경로 문제에서 BFS를 사용하는 이유는?

**1. 거리의 오름차순 탐색**
- BFS는 레벨 1, 레벨 2, 레벨 3 순으로 탐색한다. 만약 레벨 3에서 목표를 찾았다면, 레벨 1이나 2에는 해답이 없었다는 뜻이므로 '가장 먼저 찾은 경로'가 곧 '가장 짧은 경로'가 된다.

**2. 합리적인 탐색 범위**
- 모든 간선의 비용이 1로 동일할 때, 불필요하게 깊은 곳을 먼저 파고들지 않고 가까운 곳부터 샅샅이 검색하므로 최단 경로를 찾는 데 가장 최적화된 알고리즘이다.

### 📌 [Q5] DFS와 BFS를 언제 선택해야 하는가? 각각의 장단점은?

| 구분 | DFS (깊이 우선 탐색) | BFS (너비 우선 탐색) |
| :--- | :--- | :--- |
| **선택 순위** | 경로의 존재 유무, 백트래킹, 사이클 탐지 | **최단 경로**, 계층 분석, 목표가 시작점 인근일 때 |
| **주요 장점** | - 구현이 간단(재귀)<br>- 현재 세로 경로만 기억하면 되므로 메모리가 상대적으로 적게 듦 | - 최단 경로임을 100% 보장<br>- 해답이 시작점 근처라면 매우 빠름 |
| **주요 단점** | - 찾은 경로가 최단 경로라고 보장 못함<br>- 무한 깊이 시 스택 오버플로우 위험 | - 너비(폭)가 넓을수록 큐에 담기는 데이터가 많아져 메모리 부하 가능 |

### 📌 [Q6] 가중치가 있는 그래프에서는 BFS로 최단 경로를 못 구하는 이유는?

**1. "단거리 != 최소 비용"**
- BFS는 간선의 '개수'가 적은 경로를 먼저 찾는다. 하지만 가중치가 있다면, 5개의 짧은 가중치 간선을 지나는 경로(합 5)가 1개의 매우 큰 가중치 간선을 지나는 경로(합 10)보다 더 짧을 수 있다. 
- BFS는 이런 가중치 차이를 인지하지 못하고 무조건 간선 개수가 적은 쪽만 쫓게 된다.

**2. 상태 갱신의 불가능성**
- BFS는 한 번 방문한 노드를 다시 방문하지 않는다. 하지만 가중치 그래프에서는 다른 노드를 거쳐서 왔을 때 비용이 더 줄어든다면 최단 거리를 계속 업데이트해야 한다.
- **해결책**: 가중치가 있는 경우에는 **Dijkstra** 혹은 **Bellman-Ford** 같은 전용 알고리즘을 사용해야 한다.
