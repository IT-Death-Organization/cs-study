## OSI 7계층은 왜 나누어졌는가? 계층화의 장점은?

1. 추상화 (Abstraction)

복잡한 통신 과정을 기능별로 분리

→ 개발자는 특정 계층만 신경 쓰면 됨

1. 모듈화 (Modularity)

한 계층이 바뀌어도 다른 계층에 영향 최소화

예:

-   Application은 TCP/UDP가 어떻게 동작하는지 몰라도 됨
-   Transport는 물리적 케이블 종류를 몰라도 됨

1. 독립적 개발 가능

각 계층을 다른 팀/회사에서 독립적으로 개발 가능

1. 문제 진단이 쉬움

"어디 계층에서 문제인가?"를 단계별로 추적 가능

→ 네트워크 디버깅에 매우 중요

## 각 계층의 역할과 프로토콜 예시는?

7️⃣ Application Layer (응용계층)

-   사용자와 직접 상호작용
-   네트워크 서비스 제공

예:

-   HTTP
-   FTP
-   SMTP

---

6️⃣ Presentation Layer (표현계층)

-   데이터 형식 변환
-   인코딩 / 암호화 / 압축

예:

-   SSL/TLS
-   JPEG, MPEG 인코딩

---

5️⃣ Session Layer (세션계층)

-   세션 생성 / 유지 / 종료
-   체크포인트 관리

예:

-   NetBIOS Session
-   RPC

---

4️⃣ Transport Layer (전송계층)

-   종단 간 통신
-   신뢰성 / 흐름제어 / 혼잡제어

예:

-   TCP
-   UDP

---

3️⃣ Network Layer (네트워크계층)

-   IP 주소 기반 라우팅
-   경로 선택

예:

-   IP
-   ICMP
-   라우팅 프로토콜 (OSPF, RIP)

---

2️⃣ Data Link Layer (데이터링크계층)

-   MAC 주소 기반 통신
-   오류 검출

예:

-   Ethernet
-   ARP
-   PPP

---

1️⃣ Physical Layer (물리계층)

-   전기 신호, 케이블, 전압 규격

예:

-   UTP 케이블
-   광섬유

## TCP/IP 4계층과 OSI 7계층의 차이는?

계층 비교

| OSI 7계층    | TCP/IP 4계층      |
| ------------ | ----------------- |
| Application  | Application       |
| Presentation | (합쳐짐)          |
| Session      | (합쳐짐)          |
| Transport    | Transport         |
| Network      | Internet          |
| Data Link    | Network Interface |
| Physical     | Network Interface |

OSI

-   이론적 참조 모델
-   교육용
-   세분화되어 있음

TCP/IP

-   실제 구현 모델
-   단순화됨
-   인터넷 표준

## 캡슐화와 역캡슐화란 무엇이며 어떻게 동작하는가?

**캡슐화 (Encapsulation)**

데이터가 하위 계층으로 내려가며 각 계층의 **헤더(Header)** 가 붙는 과정

예시:

Application Data

↓

TCP Header 추가 → Segment

↓

IP Header 추가 → Packet

↓

Ethernet Header 추가 → Frame

↓

Physical → Bit 전송

**역캡슐화 (Decapsulation)**

수신 측에서 반대로하위 계층부터 헤더 제거

Frame

→ Packet

→ Segment

→ Data

**핵심 개념**

각 계층은:

-   자기 계층의 헤더만 해석
-   나머지는 Payload로 취급

이것이 계층 독립성의 핵심이다.

## 실제 통신에서는 어떤 모델을 사용하는가?

✔ 실제 인터넷은 **TCP/IP 4계층 모델 기반**

운영체제 네트워크 스택도 TCP/IP 기반

예:

-   Linux network stack
-   Windows TCP/IP stack

OSI는 이론적 설명 모델에 가깝다.

## 계층을 넘어서 통신할 수 없는 이유는?

### 인터페이스 계약(Contract) 때문

각 계층은:

-   위/아래 계층과 정해진 인터페이스로만 통신

Transport가 바로 Physical에 접근하면

→ 계층 구조 붕괴

### 추상화 붕괴 방지

Application이 MAC 주소를 직접 다루면?

-   유지보수 지옥
-   이식성 없음
-   복잡도 폭증

### 결합도 최소화

계층 위반은 Tight Coupling을 초래
