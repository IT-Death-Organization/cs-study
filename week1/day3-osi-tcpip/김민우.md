## 1. OSI 7계층은 왜 나누어졌는가? 계층화의 장점은?

### 계층화 이유
OSI 7계층은 복잡한 네트워크 통신을 **체계적으로 관리**하고 **표준화**하기 위해 계층으로 나누어졌습니다.

### 계층화의 장점
- **모듈화**: 각 계층이 독립적으로 동작하여 특정 계층의 변경이 다른 계층에 영향을 주지 않음
- **표준화**: 각 계층별 표준 프로토콜을 정의하여 서로 다른 시스템 간 호환성 보장
- **문제 해결 용이**: 네트워크 문제 발생 시 특정 계층을 집중적으로 분석 가능
- **개발 효율성**: 각 계층별로 독립적인 개발이 가능하여 분업과 전문화 촉진
- **유연성**: 특정 계층의 기술만 교체하거나 업그레이드 가능

## 2. 각 계층의 역할과 프로토콜 예시

### 7계층: 응용 계층 (Application Layer)
- **역할**: 사용자와 직접 상호작용하는 애플리케이션 서비스 제공
- **프로토콜**: HTTP, HTTPS, FTP, SMTP, DNS, Telnet

### 6계층: 표현 계층 (Presentation Layer)
- **역할**: 데이터 형식 변환, 암호화, 압축
- **프로토콜**: SSL/TLS, JPEG, MPEG, ASCII

### 5계층: 세션 계층 (Session Layer)
- **역할**: 통신 세션 관리 및 동기화
- **프로토콜**: NetBIOS, RPC, PPTP

### 4계층: 전송 계층 (Transport Layer)
- **역할**: 종단 간 신뢰성 있는 데이터 전송, 흐름 제어, 오류 제어
- **프로토콜**: TCP, UDP
- **단위**: Segment

### 3계층: 네트워크 계층 (Network Layer)
- **역할**: 논리적 주소(IP) 기반 경로 설정 및 라우팅
- **프로토콜**: IP, ICMP, ARP, OSPF, BGP
- **단위**: Packet

### 2계층: 데이터링크 계층 (Data Link Layer)
- **역할**: 물리적 주소(MAC) 기반 노드 간 데이터 전송, 오류 검출
- **프로토콜**: Ethernet, PPP, HDLC
- **단위**: Frame

### 1계층: 물리 계층 (Physical Layer)
- **역할**: 비트 스트림을 전기적 신호로 변환하여 물리적 매체로 전송
- **프로토콜**: RS-232, Ethernet 물리 표준
- **단위**: Bit

## 3. TCP/IP 4계층과 OSI 7계층의 차이

| OSI 7계층 | TCP/IP 4계층 | 설명 |
|-----------|--------------|------|
| 응용 계층 | 응용 계층 | OSI의 응용/표현/세션 계층을 |
| 표현 계층 | (Application) | 하나로 통합 |
| 세션 계층 | | |
| 전송 계층 | 전송 계층 (Transport) | 동일한 역할 |
| 네트워크 계층 | 인터넷 계층 (Internet) | 동일한 역할 |
| 데이터링크 계층 | 네트워크 접근 계층 | OSI의 데이터링크와 물리 계층을 |
| 물리 계층 | (Network Access) | 하나로 통합 |

### 주요 차이점
- **OSI**: 이론적 참조 모델, 7계층으로 세분화
- **TCP/IP**: 실제 인터넷에서 사용되는 실용적 모델, 4계층으로 단순화
- **개발 시기**: OSI는 후발 주자, TCP/IP가 먼저 개발되어 사실상 표준이 됨

## 4. 캡슐화와 역캡슐화란 무엇이며 어떤 동작하는가?

### 캡슐화 (Encapsulation)
송신 측에서 데이터를 전송할 때 상위 계층에서 하위 계층으로 내려가며 각 계층의 헤더(필요 시 트레일러)를 추가하는 과정입니다.

**동작 과정**:
1. **응용 계층**: 데이터 생성
2. **전송 계층**: TCP/UDP 헤더 추가 → Segment
3. **네트워크 계층**: IP 헤더 추가 → Packet
4. **데이터링크 계층**: Ethernet 헤더/트레일러 추가 → Frame
5. **물리 계층**: 비트 스트림으로 변환하여 전송

### 역캡슐화 (Decapsulation)
수신 측에서 데이터를 받을 때 하위 계층에서 상위 계층으로 올라가며 각 계층의 헤더를 제거하는 과정입니다.

**동작 과정**:
1. **물리 계층**: 비트 스트림 수신
2. **데이터링크 계층**: Frame 헤더/트레일러 제거
3. **네트워크 계층**: IP 헤더 제거
4. **전송 계층**: TCP/UDP 헤더 제거
5. **응용 계층**: 원본 데이터 전달

## 5. 실제 통신에서는 어떤 모델을 사용하는가?

실제 인터넷 통신에서는 **TCP/IP 4계층 모델**을 사용합니다.

### 이유
- TCP/IP는 실제 구현을 기반으로 설계되어 실용성이 높음
- 인터넷의 기반 프로토콜로 먼저 개발되어 사실상 표준이 됨
- OSI 7계층은 주로 **교육 및 네트워크 문제 분석**의 참조 모델로 활용됨

### 활용 방식
- **TCP/IP**: 실제 프로토콜 구현 및 통신
- **OSI**: 네트워크 개념 설명, 문제 진단, 표준화 논의

## 6. 계층을 넘어서 통신할 수 없는 이유는?

### 계층 간 통신 제약 이유

**1. 계층별 독립성 원칙**
- 각 계층은 바로 인접한 상하위 계층과만 인터페이스를 통해 통신
- 특정 계층은 자신의 역할만 수행하고 다른 계층의 세부 구현을 알 필요가 없음

**2. 추상화와 캡슐화**
- 각 계층은 하위 계층의 복잡성을 숨기고 상위 계층에 간단한 서비스만 제공
- 계층을 건너뛰면 필요한 헤더 정보나 처리 과정이 누락됨

**3. 프로토콜 스택의 순차적 처리**
- 각 계층의 헤더는 순차적으로 추가/제거되어야 함
- 중간 계층을 건너뛰면 데이터 형식이나 주소 정보가 올바르게 처리되지 않음

**4. 의존성 관계**
- 상위 계층은 하위 계층의 서비스에 의존
- 예: 응용 계층의 HTTP는 전송 계층의 TCP 없이 동작 불가

### 예외: 크로스 레이어 최적화
일부 특수한 경우 성능 향상을 위해 계층 간 정보를 공유하는 크로스 레이어 설계가 연구되지만, 이는 계층화 원칙을 일부 완화하는 특수 사례입니다.
