## 1. 해시 충돌은 왜 불가피한가? (비둘기집 원리)

**비둘기집 원리에 의한 한계**
- 무한한 키(Key) 공간을 유한한 해시 테이블 버킷 공간으로 매핑하기 때문에 발생한다.
- 비둘기집 원리에 따라 $n$개의 항목을 $m$개의 용기에 넣을 때 $n > m$이라면, 적어도 하나의 용기에는 두 개 이상의 항목이 들어갈 수밖에 없다. 실제 키 공간보다 해시 테이블 크기가 작으므로 충돌은 수학적으로 불가피하다.

---
## 2. Chaining과 Open Addressing의 장단점은?

**Chaining (체이닝)**
- **특징:** 각 버킷을 연결 리스트(또는 트리)로 구성하여 충돌된 데이터를 연결한다.
- **장점:** 해시 테이블이 꽉 차지 않아도 데이터를 계속 추가할 수 있으며 삭제가 비교적 쉽다.
- **단점:** 포인터 저장을 위한 추가 메모리가 필요하며, 한 버킷에 데이터가 쏠리면 성능이 저하된다.

**Open Addressing (개방 주소법)**
- **특징:** 충돌 발생 시 테이블 내의 다른 비어있는 공간을 찾아 저장한다.
- **장점:** 추가적인 메모리 공간(포인터)이 필요 없고 캐시 효율이 좋다.
- **단점:** 해시 테이블의 로드 팩터가 높아지면 성능이 급격히 떨어지며 삭제 처리가 복잡하다.

---
## 3. Linear Probing에서 clustering 문제는 무엇이며 왜 발생하는가?

**Primary Clustering (기본 군집화)**
- 선형 조사법(Linear Probing)은 충돌 시 바로 다음 인덱스를 확인하는 방식이다.
- 특정 영역에 데이터가 몰려 있는 '군집'이 형성되면, 새로운 데이터가 그 영역에 접근할 때마다 충돌 확률이 높아지고 빈칸을 찾기 위한 탐색 시간이 길어지는 연쇄 반응이 일어난다. 이는 충돌 시 고정된 보폭(1칸)으로만 이동하기 때문에 발생한다.

---
## 4. Load Factor가 0.75를 넘으면 왜 성능이 저하되는가?

**임계치와 성능의 상관관계**
- 로드 팩터($\alpha = \frac{n}{k}$)는 전체 버킷 중 사용 중인 버킷의 비율을 뜻한다.
- 이 값이 0.75를 넘어서면 빈 공간이 부족해져 충돌(Collision) 횟수가 지수적으로 증가하기 시작한다. 특히 Open Addressing 방식에서는 빈칸을 찾기 위한 조사(Probing) 횟수가 급격히 늘어나 $O(1)$의 성능을 유지하기 어려워진다.

---
## 5. Rehashing은 언제 발생하며 어떻게 동작하는가? 시간복잡도는?

**테이블 재구성**
- **발생 시점:** 로드 팩터가 미리 설정한 임계치(보통 0.75)를 초과할 때 발생한다.
- **동작 방식:** 기존 테이블보다 보통 2배 큰 새로운 해시 테이블을 생성하고, 기존의 모든 키를 새로운 해시 함수를 통해 다시 계산하여 새 테이블로 옮긴다.
- **시간 복잡도:** $n$개의 모든 요소를 다시 해싱하여 옮겨야 하므로 $O(n)$이 소요된다. 단, 분할 상환(Amortized) 분석 시 개별 삽입의 평균 비용은 여전히 $O(1)$로 간주한다.

---
## 6. 해시 테이블의 최악의 경우 시간복잡도가 O(n)인 이유는?

**모든 키의 충돌 (Hash Collision)**
- 사용 중인 모든 데이터가 우연히 또는 의도적으로 동일한 해시 값을 가져 단 하나의 버킷으로 몰리는 경우다.
- Chaining 방식에서는 하나의 연결 리스트를 끝까지 탐색해야 하므로 $O(n)$이 소요되며, Open Addressing 방식에서도 거의 모든 버킷을 순회해야 하므로 $O(n)$의 성능을 보이게 된다. 따라서 좋은 해시 함수를 선택하는 것이 성능의 핵심이다.
